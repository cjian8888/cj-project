# 工资识别与报告系统升级记录 (2026-01-06 16:31)

## 1. 迭代背景

在完成基础的工资识别逻辑优化后，根据用户的进一步核查需求，针对“劳务收入”、“报销剔除”等细节进行了精细化调整，并重点验证了复杂样本（朱永平）的数据准确性，最后增强了报告展示功能。

## 2. 核心功能升级

### 2.1 劳务收入全量纳入

* **调整内容**：将 `'劳务'`, `'劳务费'`, `'专家费'`, `'评审费'`, `'讲课费'`, `'稿费'` 等广义劳务关键词提升至 **强工资关键词 (Priority 1)** 级别。
* **目的**：确保低频但性质明确的劳务性收入（如偶尔一次的专家评审费）不被漏判。

### 2.2 报销剔除与豁免机制

* **剔除逻辑**：建立负面清单，强制剔除包含 `'报销'`, `'差旅'`, `'出差'`, `'交通'`, `'住宿'` 等关键词的非工资性收入（即使来自已知发薪单位）。
* **豁免保护**：建立豁免名单（`'安家费'`, `'年终奖'`, `'劳务'` 等）。如果摘要同时包含负面词（如“报销”）和豁免词（如“安家费”），系统将判定其为实质性收入予以保留。
  * *案例验证*：朱明 36万元的 “代发报销...安家费” 成功保留；马尚德 3550元的 “差旅费报销” 成功剔除。

### 2.3 报告增强

* **功能新增**：在 `核查结果分析报告.txt` 中，针对“工资性收入”字段，增加了自动化的**分年度统计明细**。
* **展示效果**：

    ```
    工资性收入: ¥2,147,900.00
      - 2022年: ¥480,400.00
      - 2023年: ¥568,500.00
      - 2024年: ¥627,800.00
      ...
    ```
