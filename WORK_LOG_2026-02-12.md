## 主要工作

- `profiles_full.json` (44MB) 包含完整数据，但代码中多处混用导致混乱
- 报告生成器 `_build_salary_income_v4` 等方法错误地使用了简化版数据

**修复内容**:
1. `investigation_report_builder.py` - 修改 `_build_salary_income_v4`，从完整数据加载
2. `family_analyzer.py` - 添加"从业单位"字段提取
3. 模板文件 - 更新字段名兼容（`yearly_salary` / `yearlySalary`）

**验证结果**:
- ✅ 年度工资表格正常显示（2006-2024 年明细）
- ✅ 从业单位信息显示（待业、新村乡新平村5队等）
- ✅ 收入来源结构分析表格正常显示
- ✅ 五维度风险评分表格正常显示

### 二、重构 profiles 数据流（重要架构改进）

**问题**: 长期以来需要同时维护 `profiles.json` 和 `profiles_full.json`，造成混淆和错误

**解决方案**: 统一数据结构，只保留一个文件

**核心改进**:

```python
# api_server.py - serialize_profiles() 函数
# 现在同时保留：
# 1. 前端需要的扁平化字段 (camelCase)
# 2. 后端需要的完整原始数据 (snake_case)

frontend_profile = {
    # 前端字段
    "entityName": name,
    "totalIncome": summary.get("total_income", 0),
    ...
    # 后端字段
    "yearly_salary": profile_dict.get("yearly_salary", {}),
    "summary": summary,
    "bank_accounts": profile_dict.get("bank_accounts", []),
    ...
}
```

**修改文件**:
1. `api_server.py` - `serialize_profiles()` 保留完整原始数据
2. `cache_manager.py` - 
   - 从 `CACHE_FILES` 移除 `profiles_full`
   - 注释掉保存逻辑
   - 修改 `load_results()` 不再依赖 `profiles_full`
3. `investigation_report_builder.py` - 移除多处冗余的降级加载逻辑
4. `scripts/regenerate_profiles_full.py` → 归档到 `archives/`
5. `debug_profiles_full.py` → 归档到 `archives/`

**效果**:
- ✅ 程序运行后**只生成 `profiles.json`**
- ✅ **不再生成 `profiles_full.json`**
- ✅ 数据流清晰统一，消除混淆
- ✅ 报告生成正常工作
