## 系统概述

本系统通过**图论算法、时序分析、机器学习、业务规则引擎**等智能算法，自动分析银行流水和线索文件，发现潜在的利益输送、隐形资产、异常资金往来等违纪违法线索，为纪检监察工作提供技术支撑。

### 适用场景

- 核查对象的银行流水全量分析
- 核查对象与涉案公司之间的资金往来追踪
- 家族资产全貌梳理
- 异常收入来源识别
- 隐蔽利益输送链路挖掘

## 模块详解

### 一、数据处理层

| 模块 | 功能 | 说明 |
|------|------|------|
| `file_categorizer.py` | 文件分类 | 自动识别数据文件类型，匹配归属实体 |
| `data_cleaner.py` | 数据清洗 | 多银行格式标准化、智能去重 |
| `data_extractor.py` | 数据提取 | 从PDF/Excel提取结构化数据 |
| `data_validator.py` | 数据验证 | 数据质量检查、交叉验证 |

### 二、分析层

| 模块 | 功能 | 说明 |
|------|------|------|
| `financial_profiler.py` | 资金画像 | 构建个人/公司的收支结构、工资识别 |
| `family_analyzer.py` | 家族分析 | 从户籍数据推导家庭关系 |
| `asset_analyzer.py` | 资产分析 | 关联房产、车辆交易记录 |
| `wealth_account_analyzer.py` | 理财账户分析 | 银行理财子账号识别与资金流转分析 |

### 三、疑点检测层

| 模块 | 功能 | 说明 |
|------|------|------|
| `suspicion_detector.py` | 核心疑点 | 现金时空伴随、直接资金往来、固定频率异常 |
| `loan_analyzer.py` | 借贷分析 | 高利贷识别、无还款借贷、规律还款模式 |
| `income_analyzer.py` | 收入分析 | 规律性非工资收入、来源不明收入、**分期受贿检测**、可信度评分 |
| `counterparty_utils.py` | 对手方工具 | 统一的对手方排除逻辑、理财产品识别（v5.1新增） |

### 四、关联分析层

| 模块 | 功能 | 说明 |
|------|------|------|
| `related_party_analyzer.py` | 关联方分析 | 关联人直接往来、第三方中转、资金闭环 |
| `fund_penetration.py` | 资金穿透 | 图论深度分析：多跳路径、闭环检测、过账通道 |
| `multi_source_correlator.py` | 多源碰撞 | 流水+出行+住宿等多源数据关联 |

### 五、智能分析层

| 模块 | 功能 | 说明 |
|------|------|------|
| `ml_analyzer.py` | 机器学习 | 统计异常检测、图社区/团伙挖掘 |
| `time_series_analyzer.py` | 时序分析 | 周期性检测、突变检测、延迟转账模式 |
| `risk_scoring.py` | 风险评分 | 交易/对手方/账户多维度量化评分 |

### 六、报告层

| 模块 | 功能 | 说明 |
|------|------|------|
| `report_generator.py` | 报告生成 | Excel底稿 + HTML报告 + 公文报告 |

| `clue_aggregator.py` | 线索聚合 | 以人员为中心的证据包聚合、综合风险评分（v5.1增强） |

### 七、增强功能层

## 快速开始

### 环境准备

```bash
# 创建虚拟环境
python -m venv .venv
source .venv/bin/activate  # macOS/Linux
# .venv\Scripts\activate   # Windows

# 安装依赖
pip install pandas openpyxl numpy streamlit pyyaml
```

### 数据准备

将待分析数据放置在项目根目录或指定目录，支持以下格式：
- 银行流水（Excel）
- 房产登记（Excel/PDF）
- 车辆登记（Excel）
- 出行记录（Excel）
- 住宿记录（Excel）

文件命名建议包含：`姓名_身份证号_数据类型.xlsx`

### 运行分析

```bash
# 默认分析当前目录数据，输出到 ./output
python main.py

# 指定数据目录和输出目录
python main.py /path/to/data /path/to/output
```

### 启动Web界面

```bash
# 启动Streamlit交互式界面
streamlit run streamlit_app.py
```

## 配置说明

### 通用性原则 ⚠️

**重要**：为保证系统的通用性和可移植性：

1. **配置文件中不包含特定案件的具体信息**（如人名、公司名）
2. **节假日自动根据核查时间区间生成**，无需手动配置
3. **案件特定配置**应在运行时通过 `USER_DEFINED_*` 参数指定

### 配置文件

配置文件 `config/default.yaml` 支持自定义：

```yaml
# === 关键词配置 ===
salary:
  strong_keywords: ['工资', '代发薪', '薪资', '绩效', '奖金']
  # 用户自定义发薪单位（运行时填写，代码库中保持为空）
  user_defined_payers: []
  
wealth_management:
  keywords: ['理财', '申购', '赎回', '添利', '安心计划']
  
cash:
  keywords: ['现金', '现取', '现存', 'ATM']

# === 节假日配置 ===
holiday:
  auto_generate: true    # 自动根据数据时间范围生成节假日
  custom_holidays: []    # 如需覆盖，可在此手动配置

# === 阈值配置 ===
thresholds:
  large_cash: 50000      # 大额现金阈值（元）
  cash_time_window_hours: 48       # 现金伴随时间窗口（小时）
  income_min_amount: 1000          # 收入分析最低金额（元）
```

### 案件特定配置

运行时可通过修改以下配置适配具体案件：

```python
# config.py 中的用户干预配置区
USER_DEFINED_SALARY_PAYERS = [
   '具体案件的发薪单位1',
   '具体案件的发薪单位2',
]
```

### 节假日自动生成

系统使用 `holiday_utils.py` 模块自动计算节假日：

```python
from holiday_utils import HolidayDetector

# 从交易数据自动创建节假日检测器
detector = HolidayDetector.from_transactions(transactions_data)

# 检查日期是否在节假日窗口
is_in_window, holiday_name, period_type = detector.is_holiday_window(date)

# 获取风险等级
risk_level = detector.get_risk_level(date, amount)
```

### 多环境配置

支持通过环境变量切换配置：

```bash
# 使用开发环境配置
export CONFIG_ENV=dev
python main.py

# 使用生产环境配置
export CONFIG_ENV=prod
python main.py
```

## 版本历史

### v5.1.0 (2026-01-11) - 纪检审计专业增强版

**🔍 审计能力增强**：基于纪检审计视角的深度代码审视，针对核心模块进行专项优化

- **分期受贿检测** (`income_analyzer.py` 新增)
  - 检测从同一个人处每月收到固定金额的可疑模式
  - 多维度风险因素评分（金额稳定性、持续时间、摘要特征）
  - 自动区分HIGH/MEDIUM风险等级
  - 排除已知发薪单位、金融机构、政府机关

- **工资识别增强** (`financial_profiler.py` 优化)
  - 第四轮高频识别增加金额上限（>10万/月需人工复核）
  - 个人姓名对手方不自动识别为工资（防止分期受贿误判）
  - 政府机关来源的规律性收入仍可正常识别

- **公共节点排除优化** (`fund_penetration.py` 优化)
  - "银行"使用精确匹配，区分纯银行与含"银行"的空壳公司
  - 正规基金公司排除，基金代理/销售公司保留
  - 支付平台使用完整匹配

- **风险评分权重优化** (`clue_aggregator.py` 优化)
  - 高风险交易：增加金额加成（每10万+5分）
  - 周期性收入：降低权重（8分→5分，减少误报）
  - 资金突变：降低权重（5分→3分，很多是理财）
  - 新增借贷关系评分（8分/个）

- **代码重构** (`counterparty_utils.py` 新建)
  - 统一对手方排除逻辑，消除6个重复函数
  - 统一理财产品识别，引入可信度分级（high/medium/low）
  - 新增 `ExclusionContext` 枚举区分不同场景

- **配置参数扩展** (`config.py` 新增30个参数)
  - 分期受贿检测参数
  - 工资识别增强参数
  - 风险评分权重参数
