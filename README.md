## 系统概述

本系统通过智能算法自动分析银行流水和线索文件，发现潜在的利益输送、隐形资产、异常资金往来等违纪违法线索，为纪检监察工作提供技术支撑。

### 核心能力

- 🔍 **多源数据融合**：银行流水、房产登记、车辆登记、出行记录、住宿记录、快递信息
- 💰 **资金穿透追踪**：核心人员↔涉案公司↔第三方账户的资金流向链条
- 👥 **家族关系分析**：基于户籍数据自动构建家族关系图谱
- ⚠️ **智能疑点检测**：6大类疑点检测算法，覆盖现金走账、利益输送、隐形资产
- 📊 **可视化报告**：HTML交互式报告 + Excel底稿 + 公文格式报告

### 一、数据处理层

#### 1.1 文件分类器 `file_categorizer.py`

**功能**：自动识别数据文件类型和归属实体

**核心算法**：

```python
def categorize_file(filepath):
    """
    1. 从文件名提取核心人员姓名/公司名称
    2. 从目录层级识别数据类型（银行、房产、车辆等）
    3. 从文件名关键词识别银行名称
    4. 返回分类结果: {entity_name, entity_type, data_type, bank_name}
    """
```

**实现细节**：

- 支持的数据类型：银行流水、房产登记、车辆登记、户籍信息、同行人、同住宿
- 使用正则表达式匹配身份证号（18位/15位）
- 银行名称识别覆盖：工商、建设、农业、中行、交通、招商、浦发、民生、光大、华夏等

### 三、疑点检测层

#### 3.1 核心疑点检测 `suspicion_detector.py`

**6大类疑点检测**：

| 序号 | 疑点类型 | 检测函数 | 核心逻辑 |
|------|---------|---------|---------|
| 1 | 现金时空伴随 | `detect_cash_timing_pattern` | A取现→B存现（48小时内，金额±5%） |
| 2 | 固定频率异常 | `detect_fixed_frequency_anomaly` | 非工资规律性收入（排除薪资、政府补贴） |
| 3 | 节假日/特殊时段 | `detect_holiday_transactions` | 节假日/周末/非工作时间大额交易 |
| 4 | 金额模式异常 | `detect_amount_patterns` | 拆分规避（接近5万）、整数偏好、吉利数尾号 |
| 5 | 隐形资产 | `detect_hidden_assets` | 大额购房/购车支出与登记不符 |
| 6 | 直接利益输送 | 详见related_party_analyzer | 核心人员间资金往来 |

#### 3.2 关联方分析器 `related_party_analyzer.py`

**功能**：分析核心人员之间的资金往来

**检测类型**：

**A. 直接资金往来**

```python
def _analyze_direct_flows():
    """
    检测核心人员之间的直接转账：
    1. 遍历每个核心人员的交易记录
    2. 查找对手方为其他核心人员的交易
    3. 记录日期、金额、方向
    """
```

**B. 第三方中转 (A→C→B)**

```python
def _detect_third_party_relay():
    """
    检测通过第三方账户中转的资金：
    
    算法：
    1. 收集A的所有支出记录
    2. 收集B的所有收入记录
    3. 寻找配对：
       - 相同中间人C（对手方一致）
       - 时间窗口：72小时内
       - 金额相近：±15%容差
       - 金额≥1万元
    
    排除规则：
    - 支付宝、微信、财付通等公共平台
    - 淘宝、京东等电商平台
    """
```

**C. 资金闭环 (A→B→C→A)**

```python
def _detect_fund_loops():
    """
    检测资金回流模式：
    
    算法：DFS深度优先搜索
    1. 构建有向图（节点=实体，边=转账）
    2. 从每个核心人员开始DFS
    3. 查找回到起点的路径（长度≤4）
    4. 要求至少涉及2个核心人员
    """
```

#### 3.3 借贷分析器 `loan_analyzer.py`

**功能**：识别借贷行为，检测高利贷和利益输送

**A. 双向资金往来**

```python
def _detect_bidirectional_flows():
    """
    检测与同一对手方既有收入又有支出的模式：
    
    条件：
    - 收入笔数≥2，支出笔数≥2
    - 单笔金额≥5000元
    
    排除：
    - 同名对手方（自己账户间转账）
    - 发薪单位
    - 理财产品对手方（避免误判买入/赎回为借贷）
    - 政府机关
    """
```

**B. 借贷配对分析**

```python
def _detect_loan_pairs():
    """
    智能匹配借入和还款记录：
    
    配对条件：
    - 还款日期 > 借入日期
    - 时间窗口 ≤ 365天
    - 还款金额 ≥ 借入金额（ratio ≥ 1.0）
    
    年化利率计算：
    interest_rate = (repay - loan) / loan × 100%
    annual_rate = interest_rate / days × 365
    
    风险判定：
    - 年化 > 36%: 超高利贷（超法律红线）
    - 年化 > 24%: 高利贷
    - 年化 < 4% 且金额≥5万: 疑似无息借贷（利益输送）
    """
```

**C. 无还款借贷检测**

```python
def _detect_no_repayment_loans():
    """
    检测大额收入但长期无还款的情况：
    
    条件：
    - 来自个人的大额收入（≥2万元）
    - 距今时间 ≥ 180天
    - 还款比例 < 50%
    
    风险含义：可能是利益输送而非真正借贷
    """
```

## 核心算法

### 1. 现金时空伴随检测算法

```
输入: 所有核心人员的交易记录
输出: 现金配对列表

for person_A in 核心人员:
    withdrawals = 提取A的现金取款记录
    
    for person_B in 核心人员 (B ≠ A):
        deposits = 提取B的现金存款记录
        
        for w in withdrawals:
            for d in deposits:
                if d.时间 - w.时间 < 48小时
                   and |d.金额 - w.金额| / w.金额 < 5%:
                    记录配对(A取现→B存现)
```

### 2. 家族关系推导算法

```
对于每个核心人员P:
    找出户主H
    确定P与H的关系 R_P
    
    对于同户每个成员M:
        if P是户主:
            M与P的关系 = M与H的关系
        else:
            M与P的关系 = 推导(R_P, M与H的关系)
            
推导规则:
    R_P="子" + M="妻" → M是P的"母"
    R_P="子" + M="子" → M是P的"兄弟姐妹"
    R_P="妻" + M="子" → M是P的"子女"
```

### 3. 借贷配对算法

```
for 每个对手方C:
    incomes = P从C收到的大额收入（可能是借入）
    expenses = P向C支付的大额支出（可能是还款）
    
    for income in incomes:
        for expense in expenses:
            if expense.日期 > income.日期
               and expense.日期 - income.日期 < 365天
               and expense.金额 >= income.金额:
               
                利率 = (expense.金额 - income.金额) / income.金额
                年化 = 利率 / 天数 * 365
                
                记录借贷配对(借入日期, 还款日期, 年化利率)
```
