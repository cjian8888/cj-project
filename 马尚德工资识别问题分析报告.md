## 二、三大核心问题

### 问题1：上海空间电源研究所（漏识别56.05万元）

**基本情况：**

- 41笔交易，跨越28个月（2022-09-25至2025-09-25）
- 总金额：56.05万元
- 多笔记录明确标注"代发工资"、"代发奖金"

**未识别原因：**

```
❌ 金额变异系数CV=3.35，远超0.8阈值
```

**问题分析：**

1. **金额构成：**
   - 常规工资：150-7955元/月
   - 年终奖金：125,626.8元（2024-12-31）
   - 大额补贴：277,200元（2024-09-30）

2. **CV值计算缺陷：**
   - 现有逻辑：`CV = 标准差 / 平均值`
   - 工资+奖金的混合导致CV值飙升
   - **但奖金、补贴本身就是工资性收入的一部分！**

3. **摘要信息被忽略：**
   - 多笔记录明确写着"代发工资"、"代发奖金"
   - 但因为之前已被高频稳定收入检测排除，后续关键词检测未执行

**改进建议：**

1. **提前进行关键词检测**：在高频稳定收入分析之前，先检查摘要是否包含"代发工资"、"代发奖金"等关键词
2. **调整CV计算策略**：
   - 方案A：识别并剔除异常值（如使用IQR方法）后再计算CV
   - 方案B：放宽CV阈值至1.5或2.0
   - 方案C：分段检测（去除最大值后重新计算CV）

## 三、数据清洗严谨性反思

### 反思1：过度依赖单一指标

- **现象**：CV\u003c0.8成为硬性门槛
- **问题**：工资+奖金是常见模式，但会被CV拒之门外
- **教训**：应该综合多个维度，而非单一指标一票否决

### 反思2：检测顺序不合理

- **现象**：先执行复杂的高频稳定收入检测，后执行简单的关键词检测
- **问题**：一旦被前者排除，后续检测就不执行了
- **教训**：应该先执行高置信度的检测（明确包含"工资"、"薪资"等），再执行推测性检测

### 反思3：摘要信息未充分利用

- **现象**：上海空间电源研究所的记录中多次出现"代发工资"、"代发奖金"
- **问题**：这些强信号被忽略了
- **教训**：应该对摘要进行深度解析，"代发工资"应该是100%的工资

### 反思4：数据质量验证不足

- **现象**：大量nan字段的记录未被标记为"数据质量异常"
- **问题**：这些记录进入分析流程，但由于缺失关键字段导致无法识别
- **教训**：应该增加数据质量报警，对缺失关键字段的记录进行特殊处理

def calculate_stable_cv(amounts, remove_outliers=True):
    """
    计算CV，可选是否剔除异常值
    """
    if remove_outliers and len(amounts) \u003e 3:
        # 使用IQR方法剔除异常值
        q1 = np.percentile(amounts, 25)
        q3 = np.percentile(amounts, 75)
        iqr = q3 - q1
        lower = q1 - 1.5 * iqr
        upper = q3 + 1.5 * iqr
        filtered = [x for x in amounts if lower \u003c= x \u003c= upper]
        
        # 如果剔除后还剩足够数据
        if len(filtered) \u003e= len(amounts) * 0.5:
            amounts = filtered
    
    mean_amt = sum(amounts) / len(amounts)
    std_amt = (sum((x - mean_amt) ** 2 for x in amounts) / len(amounts)) ** 0.5
    return std_amt / mean_amt if mean_amt \u003e 0 else 999

## 七、总结

这次分析暴露了工资识别逻辑的三大问题：

1. **过度依赖单一指标（CV值）**：导致正常的工资+奖金模式被排除

通过实施上述改进方案，可以大幅提升工资识别的准确性和完整性。

**核心教训：数据分析要更加关注业务实际，不能机械地套用统计阈值。**
