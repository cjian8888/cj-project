# 工资性收入识别改进方案

## 问题分析

### 当前识别逻辑的不足

1. **仅依赖关键词匹配**
   - 只识别明确包含"工资"、"薪资"、"奖金"等关键词的交易
   - 无法识别劳务派遣公司代发的工资（如"北京中科航天人才服务有限公司"）

2. **规律性检测过于简单**
   - 只检测金额相同的定期收入
   - 未考虑工资可能有浮动（加班费、绩效等）
   - 未排除理财利息等其他定期收入

3. **缺少对手方分析**
   - 未分析收入来源的性质
   - 未识别人力资源/劳务派遣公司

## 改进方案

### 核心思路：多维度综合判断

#### 维度1：关键词匹配（基础）

- 摘要包含：工资、薪资、薪金、奖金、津贴、补贴等
- **保持现有逻辑**

#### 维度2：对手方特征识别（新增）⭐

识别以下类型的对手方为"工资发放方"：

1. **人力资源/劳务派遣公司**
   - 包含关键词：人才服务、人力资源、劳务派遣、劳务外包、人力外包
   - 例如：北京中科航天人才服务有限公司上海分公司

2. **企业单位**
   - 排除：银行、支付平台、金融机构、个人
   - 包含：公司、集团、研究所、事业单位等
   - 需要是**高频**对手方（出现≥3次）

3. **财务/会计关键词**
   - 对手方包含：财务部、会计部、薪酬

#### 维度3：金额规律性分析（增强）⭐

1. **定期性检测**
   - 每月固定日期（±5天容差）
   - 连续出现≥3个月

2. **金额特征**
   - 金额在合理工资范围（1000-50000元）
   - 允许一定浮动（±30%），因为有加班费、绩效等

3. **排除规则**
   - 排除理财利息（金额<100元且对手方为银行）
   - 排除第三方支付小额收入
   - 排除明确标注为"借款"、"还款"的

#### 维度4：同一对手方聚类分析（新增）⭐

1. **高频对手方识别**
   - 统计所有收入对手方的频次
   - 频次≥3次的对手方标记为"潜在雇主"

2. **金额稳定性分析**
   - 计算该对手方的收入金额标准差
   - 标准差/均值 < 0.5 → 稳定收入 → 疑似工资

3. **时间分布分析**
   - 检查收入是否均匀分布在不同月份
   - 不是集中在某一两个月（排除项目款、年终奖等）

### 综合判定规则

```
工资性收入 = 满足以下任一条件：

1. 摘要明确包含工资关键词 ✅
   
2. 对手方为人力资源/劳务公司 + 金额>1000元 ✅
   
3. 对手方为企业单位 + 定期性收入（每月固定日期±5天，≥3次）+ 金额1000-50000元 ✅
   
4. 同一对手方高频收入（≥6次）+ 金额稳定（CV<0.5）+ 时间分布均匀 ✅

排除：
- 理财利息（金额<100元 + 对手方为银行 + 摘要包含"利息"）
- 第三方支付小额（金额<500元 + 对手方为支付宝/微信/财付通）
- 借款还款（摘要包含"借"、"还"、"贷"）
- 转账（对手方为个人姓名 + 摘要为"转账"）
```

## 风险控制

1. **误判风险**
   - 可能将项目款误判为工资
   - **控制措施**：要求多个条件同时满足，设置合理阈值

2. **漏判风险**
   - 可能遗漏不规律的工资（如临时工）
   - **控制措施**：保留人工审核接口，在报告中标注判定依据

3. **数据质量**
   - 对手方信息缺失
   - **控制措施**：对缺失对手方的交易，仅依赖关键词和金额规律
