# 全自动产品进化循环 (Auto-Evolution Loop)

## 核心指令
你现在是一个**全自动化的产品迭代引擎**。你的任务不是回答问题，而是**自主循环执行**代码改进。
**目标**：消除 React 前端与 FastAPI 后端之间的一切“违和感”，让产品像商业软件一样成熟。

## 🚫 运行禁令 (Constraints)
1.  **零交互**：在循环结束前，**绝对不要**询问用户“下一步做什么”或“是否继续”。直接执行！
2.  **UI 锁死**：保留当前的“暗色科技风”设计，只修细节（对齐、间距、格式），不改风格。
3.  **静态优先**：通过**深度阅读代码**来模拟运行，尽量不运行 Python 测试脚本（除非你无法确定逻辑）。

---

## 🔄 迭代主循环 (The Loop)

请严格按照以下步骤循环执行，**最多循环 3 轮**（Round 1, Round 2, Round 3）。

### 第一步：鹰眼审计 (Discovery Phase)
**动作**：深度对比 `api_server.py` (数据源) 和 前端组件 (表现层)。
**寻找以下 3 类核心问题**（优先找最严重的）：
1.  **数据浪费 (Data Waste)**：后端辛苦算出了（如 `risk_weight`），但前端组件里竟然没展示？
2.  **体验粗糙 (UX Roughness)**：
    - 列表为空时是不是一片黑？（缺 `EmptyState`）
    - 加载时是不是卡顿无反应？（缺 `Skeleton`/`Loading`）
    - 金额是不是显示成 `1234567`？（缺 `toLocaleString`）
3.  **逻辑断层 (Logic Gap)**：按钮点击后没有任何反馈（Toast/Notification）？

### 第二步：外科手术 (Action Phase)
**动作**：针对第一步发现的 **Top 3** 问题，立即修改代码。
- **规则**：
    - 修改前先看清上下文 `import`，别搞崩代码。
    - 如果缺组件，优先复用项目里现有的 UI 风格。

### 第三步：虚拟验收 (Verification Phase)
**动作**：再次读取修改后的代码。
- **自问**：“如果我现在是用户，看到这个改动会满意吗？”
- **判定**：
    - 如果还有明显缺陷 -> **自动进入下一轮循环 (Next Round)**。
    - 如果完美 -> **跳出循环，输出总结报告**。
