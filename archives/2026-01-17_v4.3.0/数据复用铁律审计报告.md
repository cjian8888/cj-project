## 📊 铁律遵循原则

### 数据复用铁律

```
┌────────────────────────────────────────────────────────────────────┐
│  原则：已经计算过的字段，下游模块应该最大程度继承和复用             │
│                                                                    │
│  源头模块（可以使用关键词）：                                      │
│  - data_cleaner.py → 生成 is_cash 列                              │
│                                                                    │
│  下游模块（必须读取已计算的列）：                                  │
│  - suspicion_detector.py ✅ 已修复                                 │
│  - risk_scoring.py ✅ 已修复                                       │
│  - financial_profiler.py ✅ 已修复                                 │
└────────────────────────────────────────────────────────────────────┘
```

### 降级策略

当 `is_cash` 列不存在时（例如处理旧数据），自动降级为关键词匹配：

```python
if 'is_cash' in df.columns:
    cash_df = df[df['is_cash'] == True]  # 优先
elif '现金' in df.columns:
    cash_df = df[df['现金'] == '是']     # 兼容 Excel 格式
else:
    # 降级：关键词匹配（最后手段）
    cash_df = df[df['description'].str.contains(...)]
```
