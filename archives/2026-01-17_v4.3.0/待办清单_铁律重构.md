## ✅ 已关闭的任务

### ~~P0: `/api/results` 接口铁律化~~ ✅ 已完成

**问题描述**：
当前 `/api/results` 返回的是内存中的 `analysis_state.results`，包含：
- `profiles` - 资金画像
- `suspicions` - 可疑交易（现金碰撞、大额现金等）
- `analysisResults` - 借贷分析、收入分析等派生数据

这些数据是在 `run_analysis` 执行时计算并缓存在内存中的，**不是从 cleaned_data 读取的**。

**影响**：
如果用户修改了 `data_cleaner.py` 的清洗规则，重新运行分析生成了新的 `cleaned_data/*.xlsx`，
但没有重新启动 `api_server.py` 或清除缓存，前端仍会显示旧的分析结果。

**解决方案**（二选一）：

#### 方案 A：派生数据持久化（推荐）
将分析结果也保存到文件，API 从文件读取：

```
output/
├── cleaned_data/           # 原始清洗数据
│   ├── 个人/*.xlsx
│   └── 公司/*.xlsx
└── analysis_cache/         # 新增：派生分析数据
    ├── profiles.json       # 资金画像
    ├── suspicions.json     # 可疑交易
    ├── loan_analysis.json  # 借贷分析
    └── income_analysis.json# 收入分析
```

修改点：
1. `run_analysis` 结束时，将分析结果保存到 `analysis_cache/*.json`
2. `/api/results` 从 `analysis_cache/` 读取，而非内存缓存
3. 添加文件时间戳校验，确保 cache 与 cleaned_data 同步

#### 方案 B：强制缓存失效
每次 API 请求时检查 cleaned_data 目录的修改时间，若有变化则重新分析。
（性能较差，不推荐）
