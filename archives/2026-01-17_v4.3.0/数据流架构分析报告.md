## 三、列名映射问题详解

**问题发现**：列名映射存在两处定义！

### 位置 1: `data_cleaner.py` (第 610-624 行)
```python
col_mapping = {
    'date': '交易时间',
    'income': '收入(元)',
    'expense': '支出(元)',
    ...
}
```

### 位置 2: `api_server.py` (第 470 行开始)
```python
INCOME_COLUMN_VARIANTS = ['收入(元)', 'income', '收入', ...]  # 硬编码变体
EXPENSE_COLUMN_VARIANTS = ['支出(元)', 'expense', '支出', ...]
```

**风险**：如果在 `data_cleaner.py` 中修改列名（如 `收入(元)` → `收入金额`），`api_server.py` 会找不到这个列！

---

## 四、优化建议

### 建议 1: 统一列名定义到 `config.py`

```python
# config.py 中新增
COLUMN_MAPPING = {
    # 内部名 → Excel 显示名
    'date': '交易时间',
    'income': '收入(元)',
    'expense': '支出(元)',
    'is_cash': '现金',
    'category': '交易分类',
    ...
}

# 变体列表（用于读取时兼容）
INCOME_COLUMN_VARIANTS = ['收入(元)', 'income', '收入']
EXPENSE_COLUMN_VARIANTS = ['支出(元)', 'expense', '支出']
```

**效果**：今后修改列名只需改 `config.py` 一处。

---

### 建议 2: 持久化 `is_salary` 列

**当前问题**：工资识别 (`is_salary`) 只在 `financial_profiler.py` 内存中计算，每次分析都要重算。

**建议**：在 `data_cleaner.py` 中增加工资识别列，持久化到 Excel。

```python
# data_cleaner.py standardize_bank_fields() 中新增
normalized['is_salary'] = False  # 初始化为 False

# 后续在 _prepare_display_data() 中调用工资识别逻辑
```

**效果**：下游模块直接读取 `is_salary` 列，无需重复计算。
