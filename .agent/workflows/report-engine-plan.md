## 一、报告全局结构规范

```
┌─────────────────────────────────────────────────────────────────┐
│                    全 局 头 部 (仅出现一次)                       │
├─────────────────────────────────────────────────────────────────┤
│  章节〇：前言                                                    │
│    ├─ 核查依据（线索来源/审批文号）                               │
│    ├─ 数据范围（银行/房产/理财/反洗钱/差旅）                      │
│    └─ 时间区间（自动检测或手动指定）                              │
├─────────────────────────────────────────────────────────────────┤
│                    单元循环体 - 个人分析                          │
│  (每个分析单元生成 Section I ~ VIII 完整章节)                     │
├─────────────────────────────────────────────────────────────────┤
│                    公司深度核查专章                               │
│  (每个涉案公司生成独立完整核查章节)                               │
├─────────────────────────────────────────────────────────────────┤
│                    公司间交叉分析专章                             │
│  (分析多公司资金闭环与关联网络)                                   │
├─────────────────────────────────────────────────────────────────┤
│                    全 局 尾 部 (仅出现一次)                       │
│  章节Z：综合研判与建议                                           │
│    ├─ 个人问题汇总 / 公司问题汇总                                │
│    ├─ 利益输送网络图                                             │
│    └─ 下一步调查建议                                             │
└─────────────────────────────────────────────────────────────────┘
```

---

## 二、分析单元定义规则

| 单元类型 | 定义 | 成员组成 | 分析方式 |
|---------|------|---------|---------|
| **核心家庭单元** | 主核查对象的经济共同体 | 本人 + 配偶 + 子女 | 资产聚合、收支合并 |
| **独立关联单元** | 与主核查对象有血亲关系但独立经济体 | 父/母/兄弟/姐妹（每人独立成一个单元） | 独立章节遍历 |

**⚠️ 铁律**：父母、兄弟姐妹**严禁**并入核心家庭单元，必须作为独立单元分别分析。

```
┌─────────────────────────────────────────────────────────────────┐
│                    GAP 收支匹配模型                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   【输入】                           【计算】                    │
│                                                                 │

│   ┌─────────────────┐               ┌─────────────────┐         │

│            │                        └────────┬────────┘         │
│            ▼                                  ▼                  │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                                                         │   │
│   │   GAP = 总支出 - 合法收入                               │   │
│   │                                                         │   │
│   │   GAP > 0  →  存在无法解释的支出来源                    │   │
│   │   GAP < 0  →  收支基本平衡（但仍需核查非工资收入）       │   │

│                                                                 │

│                                                                 │

│                                                                 │

### Section VI: 反洗钱特征识别

```
┌─────────────────────────────────────────────────────────────────┐
│                    反洗钱模式识别矩阵                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【化整为零】结构性存款                                          │
│    ├─ 特征：短期内多笔小额存入，累计金额大                        │
│    ├─ 阈值：单笔 < 5万，24小时内 ≥ 3笔，累计 > 10万              │
│    └─ 意图：规避大额交易监控                                     │
│                                                                 │
│  【即进即出】资金过境                                            │
│    ├─ 特征：资金进入后短时间内转出，金额相近                      │
│    ├─ 阈值：进出时间差 < 24小时，金额差异 < 10%                  │
│    └─ 意图：账户仅作资金中转，非真实消费                         │
│                                                                 │
│  【现金伴随】时空碰撞                                            │
│    ├─ 特征：A取现 → B存现，时间相近、金额相近                    │
│    ├─ 阈值：时间差 < 72小时，金额差异 < 20%                      │
│    └─ 意图：通过现金切断资金链追溯                               │
│                                                                 │
│  【循环流转】资金闭环                                            │
│    ├─ 特征：A → B → C → A 形成闭环                              │
│    ├─ 检测：图算法识别有向环                                     │
│    └─ 意图：制造虚假交易流水                                     │
│                                                                 │
│  【分散输送】化大为小                                            │
│    ├─ 特征：大额资金拆分后分别转给多个关联人                      │
│    ├─ 阈值：同日向 ≥ 3人转账，单笔 < 5万，累计 > 15万            │
│    └─ 意图：分散利益输送降低被发现概率                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

    │  1. 咨询费/服务费/技术费 → 占比异常高           │
    │     • 无对应合同/成果物                         │
    │     • 收款方为关联个人/皮包公司                  │
    │                                                 │
    │  2. 采购回扣模式                                │
    │     • A单位招标 → B公司中标 → C个人获利         │
    │     • 时间线碰撞（中标后短期内大额转给个人）      │
    │                                                 │
    │  3. 空转资金                                    │
    │     • A公司 → B公司 → C公司 → A公司（闭环）     │
    │     • 无实际货物/服务交付                        │
    │                                                 │
    │  4. 拆分输送                                    │
    │     • 大额资金拆分为多笔小额                     │
    │     • 分散转给多个关联个人                       │
    │                                                 │
    └─────────────────────────────────────────────────┘
```

- 资金闭环识别（A→B→C→A）
- 共同上下游供应商/客户分析

### ⚠️ 方法论：Output-First Design（输出优先设计）

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│ 1. 写样本报告 │ ──→ │ 2. 倒推数据  │ ──→ │ 3. 定义契约  │ ──→ │ 4. 实现算法  │
│  (用假数据)   │     │   需求       │     │  (JSON)     │     │  (填真数据)  │
└──────────────┘     └──────────────┘     └──────────────┘     └──────────────┘
       │                    │                    │                    │
       ▼                    ▼                    ▼                    ▼
  "报告长这样"          "需要这些字段"       "必须输出这结构"      "按契约填充"
```

**核心原则**：先有完整的样本报告，再实现算法。算法的唯一目标是填充报告所需数据。
