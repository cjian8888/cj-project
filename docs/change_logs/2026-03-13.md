# 2026-03-13 变更记录

## 变更目标
- 修复报告成品中的占位值泄漏问题，避免 `nan`、`暂无数据`、`未提供` 等内部占位直接出现在正式报告里。
- 拉齐家庭章节标题、实际分析对象和前端 HTML 报告的展示口径，避免“户主无流水但正文写成户主本人”的误导。
- 重新跑通一次完整分析链路，并通过前端实际调用的 HTML 生成接口重新产出报告。
- 修复前端启动依赖缺失问题，保证当前仓库能正常拉起开发环境。

## 核心改动摘要
- 报告数据清洗增强：
  - `investigation_report_builder.py` 新增正式报告占位值识别与清洗工具，统一清理 `nan`、`none`、`null`、`暂无数据`、`未提供`、`信息待补充` 等文本。
  - 个人身份信息、从业单位、身份证号、车辆登记日期等字段在写入报告前统一走清洗逻辑。
- 家庭章节口径修复：
  - `investigation_report_builder.py` 调整家庭章节命名逻辑，当户主无画像而家庭内其他成员有画像时，在 txt 报告中显式标记“当前分析成员”。
  - 修复配置优先模式下的家庭构建逻辑：当已存在前端筛选后的 `analysis_units` 时，不再被 `family_units_v2/family_units` 反向补回额外的零数据家庭单元。
  - 修复报告标题主体推导逻辑，避免标题和正文首个有效对象不一致。
- 报告模板修复：
  - `templates/report_v3/person_section.html` 不再渲染 `nan/暂无数据/未提供` 这类占位从业单位、职务、入职日期。
  - 车辆登记日期缺失时，HTML 统一显示为“登记时间待补充”，不再出现“于暂无数据登记购入”。
  - `templates/report_v3/report.html` 增加家庭层面的“当前可展开分析成员”展示。
- 专项报告文案修复：
  - `specialized_reports.py` 将资金闭环金额缺失提示从“总资金: 未提供”改为“闭环金额: 当前闭环结果仅输出路径与风险等级”。
- 前端依赖修复：
  - `dashboard/package.json`
  - `dashboard/package-lock.json`
  - 将 `recharts` 更新为 `^3.8.0`，并同步锁文件，配合已存在的 `vis-network` 依赖恢复前端启动能力。
