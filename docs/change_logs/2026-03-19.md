## 今日完成

### 1. 正式报告语义包继续扩展为可交付结构

- `report_view_builder.py` 将附录 A-E 从摘要视图继续扩展为 formal chapter 结构，补齐：
  - 章节标题
  - 总结段落
  - 分项明细
  - 跟进建议
- `report_dossier_builder.py` 为个人、家庭、公司卷宗补充收支缺口解释、角色标签、问题摘要和证据索引，供正式报告与前端共用
- `report_issue_engine.py` 与 `report_fact_normalizer.py` 继续收敛风险等级、问题卡字段和主报告语义视图，减少 legacy 结构与 semantic 结构之间的漂移
- 新增 `report_text_formatter.py`，统一处理：
  - 报告时间文本人类可读化
  - QA 检查标题/摘要展示
  - 语义产物 JSON 落地时的可读化转换

### 2. 报告 QA 门控与产物刷新策略加强

- `report_quality_guard.py` 新增并收紧以下检查：
  - 附录目录标题与正式章节标题一致性
  - 附录汇总数量与正文细项数量一致性
  - 风险等级口径统一
  - 公司卷宗摘要完整性
- `investigation_report_builder.py` 在落地 `report_package.json` / `report_consistency_check.*` 时补入：
  - `artifact_meta.package_generated_at`
  - `artifact_meta.source_report_generated_at`
  - `artifact_meta.qa_guard_version`
- `api_server.py` 新增 `report_package` 过期判断逻辑，当出现以下场景时会自动触发刷新：
  - QA guard 版本落后
  - 卷宗解释字段缺失
  - 正式报告语义逻辑文件更新时间晚于当前产物
  - `qa/` 目录必要产物不完整

### 3. 主报告正式内容与 legacy 文案回退规则重构

- `investigation_report_builder.py` 调整主报告 summary / next steps 合并逻辑：
  - 默认优先使用 semantic-first 主报告视图
  - 仅在 legacy 文案能被当前语义事实支撑、且 semantic 文案仍是默认占位时，才允许 legacy 文案回填
- 对“闭环/团伙/高度怀疑/明确认定”等强措辞增加语义支撑校验，避免旧综合研判直接覆盖新语义层结论
- 公司章节渲染改为优先消费 `main_report_view.company_issue_items`，确保公司摘要、角色标签、关联人/关联公司和问题要点与主报告口径一致

### 4. 前端接通统一语义包并增强静态报告导航

- `dashboard/src/components/TabContent.tsx` 扩大对 `report_package` 的消费范围，接入：
  - formal appendix 章节
  - QA 检查摘要
  - 重点对象卷宗说明
- `dashboard/src/components/NetworkGraph.tsx` 改造为可直接消费语义包中的：
  - priority board
  - dossier snapshot
  - issue catalog
  - financial gap explanation
- 图谱页新增静态正式报告预览引导，自动从报告清单中选择可预览的主报告 HTML/TXT，而不再硬编码单一文件名
- `dashboard/src/contexts/AppContext.tsx`、`dashboard/src/services/api.ts` 与 `dashboard/src/types/index.ts` 补齐语义导航、正式报告预览和重点对象配置相关类型/接口
