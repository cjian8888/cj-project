# 2026-03-12 变更记录

## 变更目标
- 统一真实收入、工资、理财、报销款等核心财务口径，降低画像与报告之间的统计分裂。
- 增强正式报告、专项报告和归集配置生成链路的稳定性与解释性。
- 清理前后端模块配置与报告访问链路中的兼容性问题。
- 补齐安全边界与回归测试，支撑当天的大规模重构落地。

## 核心改动摘要
- 资金画像与收入分类重构：
  - `financial_profiler.py` 新增姓名别名归一、家庭成员别名匹配、单位报销/业务往来款识别、真实收入剔除明细统一口径。
  - 理财赎回、退款、本人互转、家庭成员转入、保险赔付、空白字段高频入账等规则被重新梳理，真实收入与收入分类结果对齐。
  - `api_server.py` 在分析主流程中补入工资增强、真实工资、理财风险、收支匹配、个人资金特征分析结果，并将这些结果统一写入 `profiles.json`。
- 报告生成链路增强：
  - `investigation_report_builder.py` 围绕真实收入、收支匹配、报告标题主体、分类规则明细、家庭成员待补数据等口径做了系统修复。
  - `specialized_reports.py` 调整专项 txt 报告口径，避免伪造房产估值/成交价、误报微小周期性噪声、将缺失金额显示为“未提供”。
  - `templates/report_v3/person_section.html` 与 `templates/report_v3/report.html` 同步了大额转账、反洗钱、待补成员、标题主体等展示逻辑。
- 归集配置与缓存稳定性增强：
  - `report_config/primary_targets_service.py` 增加姓名归一、关系标签标准化、分析单元去重合并、自动配置快照 `primary_targets.auto.json`。
  - `cache_manager.py` 修正缓存保存日志实现，避免重复逻辑残留。
  - `api_server.py` 改为基于当前活动输入/输出目录工作，支持停止请求状态、动态时间窗口、缓存损坏降级和结果目录安全访问。
- 前端与交互链路修复：
  - `dashboard/src/contexts/AppContext.tsx`、`dashboard/src/types/index.ts`、`dashboard/src/components/Sidebar.tsx` 清理过时模块开关，统一当前分析模块集合。
  - `dashboard/src/services/api.ts` 补齐 HTML 预览响应处理与缓存信息类型。
  - `dashboard/src/components/NetworkGraph.tsx`、`dashboard/src/components/TabContent.tsx` 修复报告预览/下载地址和 Markdown 预览逻辑。
- 清理与工程化：
  - 删除 4 个历史备份文件 `personal_fund_feature_analyzer.py.bak2/.bak3/.bak4/.bak5`。
  - 补充并强化回归测试，覆盖画像口径、报告指标、归集配置和专项报告生成。
