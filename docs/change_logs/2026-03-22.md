### 7. Python 3.8 启动链兼容修复

- 根因确认：
  - 目标为 `Windows7+` 时，Python 3.8 不支持运行期直接求值 `tuple[...] / list[...] / frozenset[...]`
  - Pydantic schema 也不能在 Python 3.8 下直接解析 `list[Model]`
- 已修复文件：
  - `api_server.py`
  - `report_generator.py`
  - `investigation_report_builder.py`
  - `suspicion_engine.py`
  - `related_party_analyzer.py`
  - `report_dossier_builder.py`
  - `schemas/transaction.py`
- 修复方式：
  - 启动链关键模块统一补 `from __future__ import annotations`
  - `report_dossier_builder.py` 补齐缺失的 `Tuple` 导入
  - `schemas/transaction.py` 将 `list[Transaction]` 改回 `List[Transaction]`
- 新增回归测试：
  - `tests/test_windows_py38_compat.py`
  - 用于防止 Windows 3.8 启动链再次回退到 3.9+ 注解语法
