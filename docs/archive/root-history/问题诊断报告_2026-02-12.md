## 一、发现的核心问题

### 1.1 工资数据字段命名不匹配 【已修复】

**问题描述**:
- 数据中实际字段名: `yearly_salary` (下划线命名)
- 代码中查找的字段名: `yearlySalary` (驼峰命名)
- 导致工资数据无法被正确读取

**数据验证**:
```json
// 实际数据结构 (profiles_full.json)
{
  "yearly_salary": {  // 下划线命名
    "summary": {
      "total": 6388796.88,
      "years_count": 16
    },
    "yearly": {
      "2006": {"total": 30000.0, ...},
      "2007": {"total": 10000.0, ...}
    }
  }
}
```

**修复措施**:
```python
# 修改 _build_salary_income_v4 方法
# 原代码:
yearly_salary = profile.get("yearlySalary", {})

# 修复后:
yearly_salary = profile.get("yearlySalary") or profile.get("yearly_salary", {})
```

**修复状态**: ✅ 已修复 (investigation_report_builder.py:6177)

---

### 1.2 单位信息数据源缺失 【非代码问题】

**问题描述**:
- 户籍数据 (`family_tree`) 中确实没有 `从业单位` 字段
- 现有字段: 姓名、身份证号、与户主关系、性别、出生日期、民族、户籍地、婚姻状况、文化程度
- 缺失字段: 从业单位、籍贯、职务、入职时间

**数据验证**:
```json
// family_tree 中的实际数据
{
  "姓名": "施灵",
  "性别": "男",
  "出生日期": "1965-04-09",
  "户籍地": "...",
  // 没有 "从业单位" 字段
  // 没有 "籍贯" 字段
}
```

**结论**: 
- 这不是代码问题，是原始数据源本身不包含这些信息
- 根据架构准则，这些属于"人事档案"数据，需要外部数据源提供
- 代码中已正确标记为 `【待调取】`
