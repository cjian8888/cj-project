# 报告生成逻辑优化说明

## 优化时间
2026-01-29

## 优化文件
`/Users/serveradmin/Desktop/test/cj-project/report_generator.py`

## 优化函数
`_generate_report_conclusion(profiles, suspicions, core_persons, involved_companies, aggregator=None)`

## 优化目标
在保持报告大架构不变的前提下，大幅提升"核查结论"部分的质量，增加：
1. **风险判断的数据溯源** - 说明为什么得出特定风险结论
2. **多维度证据分析** - 收入结构、资金往来、理财行为、现金操作
3. **风险排除说明** - 明确说明排除了哪些高风险可能
4. **通用性框架** - 适用于各种核查场景，不仅限当前案例

## 主要优化内容

### 1. 多维度风险评分机制

采用 0-100 分制评分系统，涵盖四个维度：

#### 1.1 收入结构分析（0-30分）
- **工资占比评估**（10分）
  - 工资占比 ≥ 30%：得10分（正常）
  - 工资占比 < 30% 但有规律性理财收益：得8分
  - 工资占比偏低且无规律性收益：得5分

- **工资趋势稳定性**（10分）
  - 近3-5年工资稳定（变异系数 < 0.3）：得10分
  - 工资趋势基本稳定：得5分

- **收入来源明确性**（10分）
  - 收入来源明确（工资+理财）：得10分

#### 1.2 资金往来分析（0-30分）
- **家庭转账合规性**（15分）
  - 家庭成员间转账均有明确理财目的：得15分
  - 无家庭成员间异常转账：得15分
  - 部分转账需核实：得10分

- **外部大额转账**（15分）
  - 无频繁大额外部转账：得15分
  - 大额外部转账≤3笔：得10分
  - 频繁大额外部转账：得5分

#### 1.3 理财行为分析（0-20分）
- **理财申赎规律性**（10分）
  - 按月定期定存：得10分
  - 申赎基本规律：得8分
  - 无大额理财操作：得10分

- **资金空转率**（10分）
  - 空转率 < 50%（正常）：得10分
  - 空转率 50%-80%（略高但正常）：得7分
  - 空转率 ≥ 80%（需关注）：得4分

#### 1.4 现金操作分析（0-20分）
- **现金操作规模**（20分）
  - 无大额现金存取：得20分
  - 现金总额 < 10万：得18分
  - 现金总额 10-30万：得15分
  - 现金总额 30-100万：得10分
  - 现金总额 ≥ 100万：得5分

### 2. 风险等级判定

基于总分进行风险等级判定：
- **总分 0-30分**：低风险
- **总分 31-60分**：关注级
- **总分 61-100分**：高风险

### 3. 风险判断依据（数据溯源）

新增"风险判断依据"章节，详细说明每个维度的评分依据：
```
【风险判断依据】
  本结论基于以下多维度分析得出：
  1. 收入结构合理性（得分XX分）：
     • 工资占比XX.X%（符合正常范围≥30%）
     • 工资趋势稳定: XXX(2021-2023年稳定)
     • 收入来源明确（工资+理财）
  2. 资金往来合规性（得分XX分）：
     • 家庭成员间转账XXX元，均有明确理财目的
     • 无频繁大额外部转账
  3. 理财行为规范性（得分XX分）：
     • 理财申赎有规律性（按月定期定存）
     • 资金空转率XX.X%（正常范围<50%）
  4. 现金操作规范性（得分XX分）：
     • 无大额现金存取操作
```

### 4. 风险排除说明

新增"风险排除说明"章节，明确说明排除了哪些高风险可能：
```
【风险排除说明】
  经多维度分析，已排除以下高风险可能：
  ❌ 无大额不明资金转入（最大单笔转入XXX元，来源：XXX）
  ❌ 无异常现金存取（现金操作总额XXX元）
  ❌ 无频繁敏感对象往来
  ❌ 无明显资金空转套利
```

### 5. 向后兼容性

- 保持与原有 aggregator 参数的兼容性
- 如果提供了 aggregator 且有数据，优先使用统一风险评分
- 如果 aggregator 无数据，自动回退到新的多维度评分机制
- 保持 HTML 和文本报告的生成函数不变，只修改内容生成逻辑
