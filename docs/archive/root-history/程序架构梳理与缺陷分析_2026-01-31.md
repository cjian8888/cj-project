## 🌊 数据流分析

### 阶段1：数据加载

```
[输入文件]
    │
    ├─ profiles.json ──────────────→ 7个 profile
    │   ├─ 施灵、滕雳、施承天、施育
    │   └─ 3家公司
    │
    ├─ derived_data.json ──────────→ family_data
    │   ├─ large_transactions[]  → 2341条
    │   ├─ family_summary[]      → 6个家族
    │   └─ family_assets{}       → 家庭资产
    │
    └─ 数据预处理
        ├─ 识别核心人员和公司
        ├─ 按实体分组交易数据
        └─ 模拟前端选择（家庭配置）
```

**潜在缺陷**：
- ⚠️ **硬编码的前端选择**：`front_end_selection` 是硬编码的，应该从配置文件或前端获取
- ⚠️ **缺少数据验证**：没有验证 `profiles.json` 和 `derived_data.json` 的一致性
- ⚠️ **缺少数据完整性检查**：如果某些字段缺失，程序可能崩溃或产生错误结果

### 阶段2：个人资金特征分析

```
[个人数据]
    │
    ├─ profile 数据
    │   ├─ total_salary (万元) ✅ 已修复
    │   ├─ totalIncome (分)
    │   └─ totalExpense (分)
    │
    ├─ 交易数据 (cleaned_data[person_name])
    │   ├─ transaction_type (推断)
    │   ├─ amount (分)
    │   └─ description
    │
    └─ PersonalFundFeatureAnalyzer.analyze()
        │
        ├─ 1. _analyze_income_expense_match()
        │   ├─ wage_income = total_salary (万元)
        │   ├─ total_income = totalIncome / 10000 (万元)
        │   ├─ expense_mask = ["消费", "转账支出", "取现", "提现"]
        │   ├─ total_expense = sum(amount) / 10000 (万元)
        │   ├─ gap = total_expense - wage_income
        │   ├─ coverage_ratio = wage_income / total_expense * 100
        │   └─ 评分: coverage_ratio < 60% → 20分
        │
        ├─ 2. _analyze_borrowing_behavior()
        │   ├─ borrow_mask = ["借款", "贷款"]
        │   ├─ repay_mask = ["还款"]
        │   ├─ borrow_total, repay_total
        │   └─ 评分: borrow_count >= 10 → 20分
        │
        ├─ 3. _analyze_consumption_pattern()
        │   ├─ consume_mask = ["消费", "转账支出", "取现", "提现"]
        │   ├─ investment_mask = ~包含"投资|理财|基金|..."
        │   ├─ total_consumption
        │   └─ 评分: consumption_income_ratio >= 1.2 → 20分
        │
        ├─ 4. _analyze_cash_flow_pattern()
        │   ├─ transfer_mask = ["转账支出", "转账收入"]
        │   ├─ transfer_total, transfer_count
        │   └─ 评分: transfer_count >= 20 → 20分
        │
        └─ 5. _analyze_cash_operation()
            ├─ cash_mask = ["存现", "取现"]
            ├─ cash_total, cash_count
            └─ 评分: cash_count >= 5 → 15分
```

**潜在缺陷**：
- ⚠️ **金额单位不一致**：
  - `wage_income` 使用万元（来自 profiles.json）
  - `total_income` 使用万元（totalIncome/10000）
  - 交易数据需要除以 10000 转换为万元
  - **风险**：容易产生单位混淆和计算错误

- ⚠️ **消费定义过于宽泛**：
  - `consume_mask` 包括了"转账支出"、"取现"、"提现"
  - 但投资、理财、还款通过 `investment_mask` 排除
  - **问题**：其他类型的转账（如购房、医疗、教育）也被视为消费
  - **风险**：可能高估日常消费，低估真正的消费

- ⚠️ **缺少账户间转账识别**：
  - `transfer_mask` 匹配"转账支出"和"转账收入"
  - 但没有识别家庭内部转账（应该排除）
  - **风险**：家庭内部转账会被计入资金流转，增加风险评分

#### 2. 金额单位不一致
**位置**：`PersonalFundFeatureAnalyzer`
**问题**：
- `wage_income` 使用万元
- `total_income` 使用万元
- 交易数据需要除以 10000

**影响**：容易产生计算错误

**建议修复**：
在文档中明确标注每个字段的单位，并在代码中添加单位转换函数：
```python
def to_wan_yuan(amount: float, unit: str = 'fen') -> float:
    """转换为万元"""
    if unit == 'fen':
        return amount / 100000000  # 分 → 万元
    elif unit == 'yuan':
        return amount / 10000  # 元 → 万元
    return amount
```

#### 3. 缺少数据验证
**位置**：`gen_report_final.py`
**问题**：没有验证数据完整性和一致性

**影响**：可能产生错误结果或程序崩溃

**建议修复**：
```python
def validate_data(profiles, derived_data):
    """验证数据完整性"""
    errors = []
    
    # 检查 profiles 必需字段
    required_fields = ['totalIncome', 'totalExpense', 'total_salary']
    for name, profile in profiles.items():
        for field in required_fields:
            if field not in profile:
                errors.append(f"{name} 缺少字段: {field}")
    
    # 检查 derived_data 必需字段
    if 'large_transactions' not in derived_data:
        errors.append("derived_data 缺少 large_transactions")
    
    return errors

# 在加载数据后调用
validation_errors = validate_data(profiles, derived_data)
if validation_errors:
    print(f"⚠️ 数据验证失败: {validation_errors}")
```

### 🟡 中优先级缺陷

#### 4. 消费定义过于宽泛
**位置**：`PersonalFundFeatureAnalyzer._analyze_consumption_pattern()`
**问题**：转账支出、取现都被视为消费

**建议优化**：
```python
# 更精确的消费定义
consume_mask = transactions["transaction_type"] == "消费"

# 添加转账分类
transfer_categories = {
    "购房": ["购房", "买房", "房产", "中介"],
    "医疗": ["医院", "诊所", "医药", "医疗"],
    "教育": ["学校", "培训", "教育", "学费"],
    "购车": ["4S店", "汽车", "购车", "车辆"]
}

# 分类大额转账
for category, keywords in transfer_categories.items():
    category_mask = transactions["description"].str.contains(
        "|".join(keywords), na=False, case=False
    )
    # 单独统计，不视为日常消费
```

#### 5. 缺少家庭内部转账识别
**位置**：`PersonalFundFeatureAnalyzer._analyze_cash_flow_pattern()`
**问题**：家庭内部转账被计入资金流转

**建议优化**：
```python
# 识别家庭内部转账
family_transfer_mask = (
    transactions["transaction_type"] == "转账支出" &
    transactions["counterparty"].isin(family_members)
)

# 排除家庭内部转账
external_transfer = transactions[~family_transfer_mask]

# 只计算外部转账
transfer_total = external_transfer["amount"].sum() / 10000
```
