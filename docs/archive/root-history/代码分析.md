### 2.3 问题分析

#### 问题1：v5只是v4的包装

`build_report_v5` 函数（line 4227）实际上只是调用了 `build_report_v4`，然后添加了四部分架构的外层字段：

```python
def build_report_v5(...):
    # 复用v4.0的核心逻辑构建基础数据
    base_report = self.build_report_v4(...)  # ← 实际构建还是v4
    
    # v5.0增强：构建完整四部分架构
    report = {
        # v4.0 兼容字段
        "meta": ...,
        "family_sections": base_report.get("family_sections"),
        "person_sections": base_report.get("person_sections"),
        
        # v5.0 新增：只是包装，没有新数据
        "part_a_family_analysis": {
            "family_sections": base_report.get("family_sections"),
            "summary": ...,
        },
        "part_b_person_analysis": {
            "person_sections": base_report.get("person_sections"),
        },
        ...
    }
```

#### 问题2：前端调用正确，但渲染错误

**前端调用链**:
1. `ReportBuilder.tsx` 调用 `/api/investigation-report/generate-with-config`
2. API调用 `builder.build_report_v5()`
3. v5调用 `build_report_v4()`
4. v4调用 `build_v4_person_section()` 构建个人章节

**问题**：前端 `renderReportToHtml` 函数期望的数据结构和实际返回的不匹配

#### 问题3：具体数据字段不匹配

| 准则要求 | v4实际结构 | 前端期望 | 匹配度 |
|----------|-----------|----------|--------|
| 银行存款 | `bank_deposits.bank_count` | `assets.bank_account_count` | ❌ 字段名不同 |
| 银行卡列表 | `bank_deposits.accounts` | `assets.bank_accounts` | ✅ 但路径不同 |
| 总收入 | `profile.totalIncome` | `asset_income_section.total_income` | ❌ 不存在 |
| 总支出 | `profile.totalExpense` | `asset_income_section.total_expense` | ❌ 不存在 |

## 三、为什么没有被正确调用

### 3.1 后端层面

**实际调用链是正确的**：
- 前端 → `/api/investigation-report/generate-with-config` → `build_report_v5` → `build_report_v4` → `build_v4_person_section`

**问题**：`build_v4_person_section` 构建的数据结构虽然完整，但：
1. 缺少直接从profile获取的 `total_income` / `total_expense`
2. `family_summary` 中的银行存款汇总计算错误（为0）

### 3.2 前端层面

**问题1**：`renderReportToHtml` 函数（line 213）使用了错误的数据路径：

```typescript
// 前端代码（错误）
const memberDetails = personSections.map((p: any) => ({
    total_income: assetSection.total_income || ...,  // ❌ assetSection没有total_income
    assets: {
        bank_account_count: assetSection.bank_deposits?.count || ...,  // ❌ 应该是bank_count
    }
}))
```

**问题2**：前端应该使用 `bank_deposits.bank_count` 而不是 `bank_deposits.count`

## 四、解决方案建议

### 方案1：修复前端渲染函数（推荐，最快）

修改 `ReportBuilder.tsx` 中的 `renderReportToHtml`，使用正确的数据路径：

```typescript
const assetSection = p.asset_income_section || {};
const bankDeposits = assetSection.bank_deposits || {};

return {
    name: p.name,
    relation: p.relation,
    // 从profile获取总收入/总支出（因为asset_section没有）
    total_income: p._profile_total_income || 0,  // 需要后端添加
    total_expense: p._profile_total_expense || 0,
    assets: {
        bank_accounts: bankDeposits.accounts || [],
        bank_account_count: bankDeposits.bank_count || 0,  // ✅ 修正
    }
}
```

### 方案2：修复后端 build_v4_person_section

在 `build_v4_person_section` 中添加缺失的字段：

```python
profile = self.profiles.get(name, {})

section = {
    "name": name,
    "relation": relation,
    # 添加直接从profile获取的汇总数据
    "total_income": profile.get("totalIncome", 0),  # 添加
    "total_expense": profile.get("totalExpense", 0),  # 添加
    "transaction_count": profile.get("transactionCount", 0),  # 添加
    "asset_income_section": {
        ...
    }
}
```

### 方案3：修复 family_summary 计算

修复 `_build_family_summary_v4` 函数，正确汇总银行存款：

```python
def _build_family_summary_v4(self, anchor: str, members: List[str]) -> Dict:
    ...
    total_bank_balance = 0.0
    for member in members:
        profile = self.profiles.get(member, {})
        # 从bank_deposits获取而不是bank_accounts_official
        bank_deposits = self._build_bank_deposits_v4(member, profile)
        total_bank_balance += bank_deposits.get("total_balance", 0)
```

## 五、结论

1. **最接近准则的函数**：`build_v4_person_section`（line 5953），完成度约85%
2. **为什么没有被正确调用**：实际上被调用了，但前端渲染函数使用了错误的数据路径
3. **核心问题**：前后端数据字段命名不一致
4. **修复优先级**：
   - P0: 修复前端 `renderReportToHtml` 使用正确的字段路径
   - P1: 修复后端 `family_summary` 银行存款汇总计算
   - P2: 在 `build_v4_person_section` 中添加 `total_income` / `total_expense` 字段
