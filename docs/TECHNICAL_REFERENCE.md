纪检监察审计专用的自动化资金追踪与异常检测系统，用于内部廉洁核查案件的数据分析。

## 系统概述

本系统通过智能算法自动分析银行流水和线索文件，发现潜在的利益输送、隐形资产、异常资金往来等违纪违法线索，为纪检监察工作提供技术支撑。

## 疑点检测算法详解

### 1. 现金时空伴随检测

**文件**: `suspicion_detector.py` → `detect_cash_timing_pattern()`

**检测逻辑**:

```
输入: 核心人员的现金交易记录
处理:
  1. 提取所有现金取款记录（关键词：取现、ATM、柜台取款等）
  2. 提取所有现金存款记录（关键词：存现、柜台存款等）
  3. 配对检测:
     - 时间窗口: 取现后48小时内有存现
     - 金额匹配: 金额差异在±5%以内
     - 不同主体: 取现方和存现方不是同一人
输出: 现金配对列表
```

**风险等级判定**:

| 条件 | 风险等级 |
|------|---------|
| 24小时内 + 金额差异<1% | 高风险 |
| 48小时内 | 中风险 |
| 其他 | 低风险 |

**排除规则**:

- 同一人的自我转账
- 金额<5000元的小额交易

---

### 2. 固定频率异常检测

**文件**: `suspicion_detector.py` → `detect_fixed_frequency_anomaly()`

**检测逻辑**:

```
输入: 个人交易记录
处理:
  1. 筛选收入交易（income > 0）
  2. 排除工资性收入（包含工资/代发/薪资等关键词）
  3. 排除已知发薪单位（研究所、人力资源公司等）
  4. 按金额分组（±10%容差）
  5. 检查时间规律:
     - 是否每月固定日期（±3天容差）
     - 至少出现3次
输出: 固定频率收入列表
```

**排除规则**:

- 包含工资/代发/薪资/绩效/奖金等关键词
- 来自已知发薪单位
- 金额<100元
- 社保/公积金相关

---

### 3. 节假日/特殊时段交易检测

**文件**: `suspicion_detector.py` → `detect_holiday_transactions()`

**检测逻辑**:

```
输入: 个人交易记录
处理:
  1. 提取法定节假日交易（春节、国庆等）
  2. 提取周末交易
  3. 提取非工作时间交易（晚8点-早8点）
  4. 筛选大额交易（≥5万元）
输出: 特殊时段大额交易列表
```

**配置参数**:

```python
CHINESE_HOLIDAYS = {...}           # 法定节假日配置
NON_WORKING_HOURS_START = 20       # 晚8点
NON_WORKING_HOURS_END = 8          # 早8点
HOLIDAY_LARGE_AMOUNT_THRESHOLD = 50000  # 5万元
```

---

### 4. 金额模式异常检测

**文件**: `suspicion_detector.py` → `detect_amount_patterns()`

**检测逻辑**:

```
检测类型1: 拆分规避
  - 连续多笔接近5万元的交易（4.9万-5万）
  - 检测规避大额报告制度的行为

检测类型2: 整数偏好
  - 大量整数金额交易（1万、2万、5万等）
  - 统计整数金额占比

检测类型3: 吉利数尾号
  - 金额尾号为88、66、888、666、168等
  - 可能涉及送礼行为
```

**配置参数**:

```python
SPLIT_AMOUNT_THRESHOLD = 50000    # 拆分规避阈值
SPLIT_DETECTION_COUNT = 3         # 连续笔数
ROUND_AMOUNT_THRESHOLD = 10000    # 整数金额阈值
LUCKY_TAIL_NUMBERS = ['88', '66', '888', '666', '168']
```

### 5. 隐形资产识别

**文件**: `suspicion_detector.py` → `detect_hidden_assets()`

**检测逻辑**:

```
购房检测:
  - 金额 ≥ 10万元
  - 包含关键词: 房款、首付、按揭、房地产、置业等
  - 排除: 设备、工程、材料等工业用途

购车检测:
  - 金额 ≥ 5万元
  - 包含关键词: 车款、4S店、汽车销售、奔驰、宝马等
  - 排除: 公务用车、公司采购
```

**与登记信息交叉验证**:

```
1. 读取房产登记信息（自然资源部数据）
2. 读取车辆登记信息（公安部机动车数据）
3. 匹配流水中的大额支出与登记信息
4. 验证资金来源的合理性
```

### 6. 直接利益输送检测

**文件**: `related_party_analyzer.py`

**检测逻辑**:

```
检测类型1: 直接资金往来
  - 核心人员之间的直接转账记录
  - 金额≥1万元

检测类型2: 第三方中转
  - A→C→B模式（72小时内）
  - 排除支付宝/微信等公共平台
  - 金额≥1万元

检测类型3: 资金闭环
  - A→B→C→A等资金回流模式
  - 要求至少2个不同核心人员

检测类型4: 高频中间人
  - 多次出现的中转账户
  - 可能是"掮客"或"白手套"
```

**排除规则**:

```python
THIRD_PARTY_PAYMENT_KEYWORDS = [
    '支付宝', '微信', '财付通', '京东支付',
    '淘宝', '天猫', '美团', '滴滴'
]
```

### 7. 借贷行为分析

**文件**: `loan_analyzer.py`

#### 7.1 双向资金往来检测

**检测逻辑**:

```
输入: 个人交易记录
处理:
  1. 按对手方分组
  2. 统计每个对手方的收入和支出
  3. 筛选双向往来:
     - 既有收入又有支出
     - 收入≥2笔，支出≥2笔
     - 金额≥5000元
输出: 双向往来关系列表
```

**排除规则**:

- 同名对手方（自己给自己的转账）
- 发薪单位
- 第三方支付平台

#### 7.2 借贷配对分析

**检测逻辑**:

```
输入: 个人交易记录
处理:
  1. 识别来自个人的收入（可能是借入）
  2. 识别向同一人的支出（可能是还款）
  3. 配对匹配:
     - 还款日期 > 借入日期
     - 时间窗口 ≤ 365天
     - 还款金额 ≥ 借入金额（ratio ≥ 1.0）
  4. 计算年化利率:
     利率 = (还款 - 借入) / 借入 × 100%
     年化 = 利率 / 天数 × 365
输出: 借贷配对列表
```

**风险等级判定**:

| 条件 | 风险等级 |
|------|---------|
| 年化利率 > 36% | 高风险（超法律红线） |
| 年化利率 < 0.1% 且金额≥5万 | 高风险（疑似无息借贷） |
| 借贷金额 ≥ 10万 | 高风险 |

**排除规则**:

- 同名对手方（自己账户间转账）
- 公司/机构对手方
- 发薪单位
- 银行系统账户（代付/代收/业务专户）
- 还款 < 借入（这是转账，不是借贷）

#### 7.3 无还款借贷检测

**检测逻辑**:

```
输入: 个人交易记录
处理:
  1. 识别来自个人的大额收入（≥2万元）
  2. 检查该对手方后续是否有还款
  3. 条件:
     - 距今时间 ≥ 180天
     - 还款比例 < 50%
输出: 无还款借贷列表（疑似利益输送）
```

**排除规则**:

- 同名对手方
- 发薪单位（工资不需要还款）
- 研究所/研究院（单位收入）
- 银行系统账户

#### 7.4 异常利息检测

**检测逻辑**:

```
输入: 借贷配对列表
处理:
  判断年化利率是否异常:
  - 高利贷: 年化 > 24%
  - 超高利贷: 年化 > 36%（超法律红线）
  - 无息借贷: 年化 < 4% 且金额≥5万（疑似利益输送）
输出: 异常利息列表
```

#### 7.5 借贷网络分析

**检测逻辑**:

```
输入: 借贷配对列表
处理:
  1. 构建借贷关系图谱
     - 节点: 核心人员 + 关联对手方
     - 边: 借贷关系（带金额和次数）
  2. 识别中心节点（Hub）
     - 与3人及以上有借贷关系
     - 可能是"掮客"角色
输出: 借贷网络数据
```

### 8. 多源数据交叉碰撞

**文件**: `multi_source_correlator.py`

#### 8.1 出行同行人碰撞

**检测逻辑**:

```
输入: 中航信航班数据 + 铁路总公司数据
处理:
  1. 提取核心人员的出行记录
  2. 识别同行人（同航班/同车次）
  3. 核查同行人是否与核心人员有资金往来
```

#### 8.2 同住宿人碰撞

**检测逻辑**:

```
输入: 公安部住宿信息
处理:
  1. 提取核心人员的住宿记录
  2. 识别同住人
  3. 核查同住人是否与核心人员有资金往来
```

#### 8.3 快递联系人碰撞

**检测逻辑**:

```
输入: 国家邮政局快递信息
处理:
  1. 提取快递收发记录
  2. 识别频繁联系人
  3. 核查联系人是否与核心人员有资金往来
```

---

### 9. 异常收入来源检测

**文件**: `income_analyzer.py`

#### 9.1 规律性非工资收入

**检测逻辑**:

```
输入: 个人收入记录
处理:
  1. 排除工资性收入
  2. 检测规律性收入:
     - 每月固定日期（±3天）
     - 金额稳定（变异系数<0.3）
     - 出现≥3次
输出: 可疑规律收入列表
```

#### 9.2 个人大额转入

**检测逻辑**:

```
输入: 个人收入记录
处理:
  1. 识别来自个人的大额转入（≥2万元）
  2. 排除核心人员间互转
  3. 排除已知亲属
输出: 个人大额转入列表
```

#### 9.3 来源不明收入

**检测逻辑**:

```
输入: 个人收入记录
处理:
  1. 识别对手方信息为空或模糊的大额收入
  2. 金额≥1万元
输出: 来源不明收入列表
```
