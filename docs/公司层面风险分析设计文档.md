# 公司层面风险分析模块 - 设计文档

## 一、概述

### 1.1 背景与目标

本模块用于资金穿透与关联排查系统中"公司资金核查"部分，提供深度、通用性的公司层面风险分析能力。

**核心目标：**
1. 检测公司之间的利益输送
2. 检测公司向个人的资金输送（洗钱风险）
3. 检测隐藏资产和异常支出
4. 评估公司经营合理性
5. 给出明确的风险判断和证据

### 1.2 设计原则

- **通用性**：不针对特定案例硬编码，适用于各种公司规模和行业
- **可配置性**：使用可配置的阈值（如大额转账阈值、高频阈值）
- **可审计性**：逻辑清晰，证据完整，便于审计人员理解
- **可扩展性**：模块化设计，易于增加新的分析维度

## 二、系统架构


## 三、分析维度详解

### 3.1 维度1：公司间资金往来分析（0-30分）

#### 3.1.1 频繁转账识别（0-10分）

**检测逻辑：**
- 统计公司间的转账频率和金额
- 识别每月高频转账（默认≥4笔/月）
- 计算金额变异系数（CV），判断是否为规律性转账
- CV < 0.3 视为规律性转账

**评分规则：**
- 每发现一对规律性频繁转账：+2分（最高10分）

**证据格式：**
```python
{
    "type": "频繁公司间转账",
    "company_a": "公司A",
    "company_b": "公司B",
    "count": 15,
    "total_amount": 5000000,
    "avg_amount": 333333,
    "cv": 0.15,
    "is_routine": true,
    "description": "公司A与公司B间发生15笔转账，总计500万元...",
    "strength": "强/中/弱",
    "risk_reason": "规律性频繁转账，可能涉及利益输送"
}
```

#### 3.1.2 资金流向分析（0-10分）

**检测逻辑：**
- 构建有向图表示资金流转
- 检测多跳路径（A→B→C）
- 判断中间节点是否为中转通道
- 金额相似度在0.8-1.2之间视为可疑

**评分规则：**
- 每发现一条可疑多跳路径：+3分（最高10分）

#### 3.1.3 利益输送路径识别（0-5分）

**检测逻辑：**
- 检测资金闭环（A→B→A）
- 关注短期闭环（30天内）
- 分析金额关系

**评分规则：**
- 每发现一个短期闭环：+5分（最高5分）

#### 3.1.4 空壳公司识别（0-5分）

**检测逻辑：**
- 计算进出账比例（出账/进账）
- 进出比在0.8-1.2之间且金额较大
- 主要交易对手为公司（>50%）

**评分规则：**
- 每发现一个疑似空壳公司：+5分（最高5分）

---

### 3.2 维度2：公司向个人资金输送分析（0-30分）

#### 3.2.1 大额个人转账检测（0-10分）

**检测逻辑：**
- 识别单笔超过阈值的个人转账（默认5万元）
- 识别累计超过阈值的个人转账（默认20万元）
- 区分核心人员和非核心人员

**评分规则：**
- 向核心人员大额转账：每笔+5分
- 向非核心人员大额转账：每笔+3分

#### 3.2.2 高频个人转账（0-10分）

**检测逻辑：**
- 识别向同一人员高频转账（默认≥6笔）
- 关注来自多个公司的情况

**评分规则：**
- 多公司向同一人高频转账：每个案例+5分
- 单公司高频转账：每个案例+3分

#### 3.2.3 资金分散模式（0-10分）

**检测逻辑：**
- 识别同一时间段内向多个个人账户转账
- 金额相对接近（拆分洗钱特征）
- 个人账户数≥3，转账笔数≥5

**评分规则：**
- 每发现一个资金分散模式：+5分（最高10分）

---

### 3.3 维度3：公司资产异常分析（0-20分）

#### 3.3.1 无业务大额支出（0-8分）

**检测逻辑：**
- 识别大额支出但对手方模糊
- 摘要不明确或缺失
- 排除公司、银行等正常对手方

**评分规则：**
- 每发现一笔：+2分（最高8分）

#### 3.3.2 异常费用模式（0-6分）

**检测逻辑：**
- 识别咨询费、服务费等可疑费用
- 频繁（≥3笔）或大额（>30万元）的情况

**评分规则：**
- 每发现一个案例：+2分（最高6分）

#### 3.3.3 固定资产异常（0-6分）

**检测逻辑：**
- 支出远超收入（支出/收入 > 1.5）
- 收入远超支出（支出/收入 < 0.5）

**评分规则：**
- 每发现一个异常：+3分（最高6分）

---

### 3.4 维度4：公司经营合理性分析（0-20分）

#### 3.4.1 收支匹配度（0-8分）

**检测逻辑：**
- 计算收支比（支出/收入）
- 比例>1.5或<0.5视为异常

**评分规则：**
- 比例>2：+5分
- 1.5<比例≤2：+3分
- 比例<0.5：+3分

#### 3.4.2 现金操作异常（0-6分）

**检测逻辑：**
- 识别大额现金存取（>5万元）
- 累计现金操作金额

**评分规则：**
- 总计>25万元：+4分
- 5万<总计≤25万元：+2分

#### 3.4.3 异常时间模式（0-6分）

**检测逻辑：**
- 非工作时间（非9:00-18:00）的大额交易
- 周末/节假日的大额交易
- 至少3笔异常时间交易

**评分规则：**
- 每发现一个案例：+2分（最高6分）

## 四、风险评分机制

### 4.1 总分计算

```
总分 = 公司间风险(0-30) + 公司向个人风险(0-30) + 资产异常风险(0-20) + 经营合理性风险(0-20)
```

**总分范围：0-100分**

### 4.2 风险等级划分

| 总分范围 | 风险等级 | 说明 |
|---------|---------|------|
| 0-30分 | 低风险 | 基本正常，无明显风险 |
| 31-60分 | 关注级 | 存在风险点，需关注 |
| 61-100分 | 高风险 | 严重风险，需重点核查 |

### 4.3 证据强度定义

| 强度 | 说明 |
|-----|------|
| 强 | 明显违规或高度可疑的证据 |
| 中 | 存在异常，需进一步核实 |
| 弱 | 轻微异常，仅供参考 |

### 6.2 集成到报告生成

```python
# 在 report_generator.py 中集成

from company_risk_analyzer import analyze_company_risk

def _generate_company_section(company, profiles, core_persons, cleaned_data, companies_profiles, company_transactions, core_persons_list, suspicions):
    """
    生成公司资金核查部分（增强版）
    """
    report_lines = []
    
    # ... 原有的公司分析 ...
    
    # 执行公司风险分析
    # 提取相关公司
    involved_companies = [company]
    
    risk_result = analyze_company_risk(
        companies_profiles=companies_profiles,
        company_transactions=company_transactions,
        core_persons=core_persons_list,
        suspicions=suspicions
    )
    
    # 添加风险分析结果
    report_lines.append(f"  【风险评级】: {risk_result['overall_risk_level']} ({risk_result['overall_risk_score']}分)")

    for dim_name, dim_data in risk_result["dimensions"].items():
        if dim_data["score"] > 0:
            dim_cn_name = {
                "inter_company_risk": "公司间往来",
                "company_to_person_risk": "公司向个人",
                "asset_anomaly_risk": "资产异常",

            }.get(dim_name, dim_name)
            report_lines.append(f"  • {dim_cn_name}: {dim_data['score']}分")
    
    # 风险排除说明
    if risk_result["risk_exclusions"]:
        report_lines.append("")
        report_lines.append("  【风险排除说明】")
        for exclusion in risk_result["risk_exclusions"]:
            report_lines.append(f"    ✓ {exclusion}")
    
    # 高风险红旗
    if risk_result["red_flags"]:
        report_lines.append("")
        report_lines.append("  【高风险预警】")
        for flag in risk_result["red_flags"]:
            report_lines.append(f"    ⚠ [{flag['type']}] {flag['details']}")
    
    return report_lines
```

## 十、注意事项

### 10.1 数据质量要求

- 交易数据需要包含：日期、对手方、金额、摘要等字段
- 日期字段应为可解析的日期格式
- 对手方字段需要完整，避免过多空值
- 建议预处理对手方名称（标准化）

### 10.2 误报控制

- 本模块提供风险线索，不代表最终定性
- 证据强度"中"的需人工核实
- 结合业务背景综合判断
- 注意行业差异性（如现金密集型行业）

### 10.3 性能优化

- 大数据量时考虑分批处理
- 对高频查询建立索引
- 缓存中间结果
- 并行化独立的分析维度
