# 初查报告生成准则

> 本文档定义了穿云审计系统生成初查报告的标准框架和分析维度。

---

## 一、核查流程与报告逻辑

### 1.1 两轮核查流程

```
第一轮核查：只有核查对象本人
┌─────────────┐
│   甲某某     │  ← 只有被查人的银行流水
│  (核查对象)  │
└─────────────┘
      │
      ▼ 系统自动识别（family_analyzer.py）
      │
    通过同户人/户籍人口数据发现家庭成员：
    - 乙某某（配偶）
    - 甲小某（子女）
    - 甲大某（父亲）
      │
      ▼ 发起第二轮核查（补充调取家属数据）

第二轮核查：家庭完整数据
┌─────────────────────────────────────────┐
│         甲某某 家庭                        │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌──────┐      │
│  │甲某某  │ │乙某某  │ │甲小某│ │甲大某 │      │
│  │本人  │ │配偶  │ │子女  │ │父亲   │      │
│  └─────┘ └─────┘ └─────┘ └──────┘      │
│                                          │
│  → 现在可以做家庭汇总分析               │
└─────────────────────────────────────────┘
```

### 1.2 报告生成逻辑

- **第一轮（仅本人）**：生成个人报告
- **第二轮（有家属）**：自动升级为**家庭报告**，以家庭为单位汇总，再细分个人

### 1.3 相关模块

| 模块 | 功能 |
|------|------|
| `family_analyzer.py` | 从同户人/户籍人口数据识别家庭成员 |
| `family_finance.py` | 计算家庭资产汇总、家庭成员间转账 |
| `build_family_tree()` | 构建家族关系图谱 |
| `get_family_summary()` | 生成家族关系摘要（配偶/子女/父母） |

## 二、报告整体结构

```
初查报告
├── 元信息 (meta)
│   ├── 文号 (doc_number)
│   ├── 案件背景 (case_background)
│   └── 数据范围 (data_scope)
│
├── 核查对象家庭 (family)  ← 以家庭为单位
│   │
│   ├── 一、家庭概况（汇总）
│   │   ├── 家庭成员构成
│   │   ├── 家庭资产汇总（房、车、存款）
│   │   └── 家庭收支总览
│   │
│   ├── 二、甲某某（核查对象 - 本人）
│   │   ├── 板块一：个人资产收入
│   │   └── 板块二：数据分析
│   │
│   ├── 三、乙某某（配偶）
│   │   └── ...
│   │
│   └── 四、甲小某（子女）
│       └── ...
│
└── 综合研判 (conclusion)
    ├── 研判意见 (summary_text)
    └── 下一步工作建议 (next_steps[])
```

## 三、个人分析维度

### 板块一：家庭资产及基本收入（静态盘点）

**目的**：建立被核查人的基础经济状况模型

| 维度 | 具体内容 | 数据来源 | 实现状态 |
|------|----------|----------|----------|
| **基本身份** | 出生日期、籍贯、职务、家庭成员信息 | 外部数据 | 🔲 预留 |
| **房产情况** | 地址、面积、金额、登记时间、资金来源 | 不动产数据 | 🔲 预留 |
| **工资奖金** | 税后工资、补贴、年平均收入 | 银行流水 | ✅ 已实现 |
| **银行存款** | 真实银行卡统计（见下方详细说明） | 银行流水 | ⚠️ 需完善 |
| **理财情况** | 理财产品持有量、购买/赎回记录 | 银行流水 | ✅ 已实现 |
| **车辆情况** | 品牌、车牌、购买时间、价格 | 外部数据 | 🔲 预留 |

#### 银行存款统计规则

**统计范围**：
- ✅ 真实银行卡（借记卡、储蓄卡、工资卡）
- ✅ 信用卡（如有）
- ❌ 基金账户（剔除）
- ❌ 理财子账号（剔除）
- ❌ 证券账户（剔除）

**剔除判断逻辑**：
- 账号长度异常（非16-19位）
- 账号包含"基金"、"理财"、"证券"等关键词
- 开户行为"XX基金"、"XX证券"

**表格输出格式**：

| 序号 | 银行名称 | 卡号（完整） | 账户类别 | 卡类型 | 账户状态 | 当前余额 |
|------|----------|-------------|----------|--------|----------|----------|
| 1 | 中国建设银行 | 6227XXXXXXXXXXXX | 个人账户 | 借记卡 | 正常 | ¥XX,XXX.XX |
| 2 | 交通银行 | 6222XXXXXXXXXXXX | 个人账户 | 工资卡 | 正常 | ¥XX,XXX.XX |

> **账户类别**：个人账户 / 对公账户 / 联名账户

### 板块二：交易数据深度分析（动态排查）

**目的**：通过银行流水排查异常资金往来和潜在违规行为

| 分析维度 | 具体内容 | 判断逻辑 | 实现状态 |
|----------|----------|----------|----------|
| **收支匹配分析** | 收支是否平衡、工资占比 | 工资收入/总收入 < 50% 为可疑 | ✅ 已实现 |
| **资金流入分析** | 剔除互转后的主要来源 | 按对手方分组统计 | ✅ 已实现 |
| **资金流出分析** | 主要去向、大额收款方 | 按对手方分组统计 | ✅ 已实现 |
| **大额现金分析** | 现金存取总额、异常现金 | 单笔 ≥ 阈值的现金交易 | ✅ 已实现 |
| **大额转账分析** | 阈值筛查、逐笔核查 | 单笔 ≥ 1万元 | ✅ 已实现 |
| **关联交易排查** | 与涉案公司/敏感人员往来 | 交叉比对核心人员与涉案公司 | ⚠️ 需增强 |
| **企业登记分析** | 名下企业、纳税记录 | 外部数据 | 🔲 预留 |
| **反洗钱数据** | 可疑交易监测 | 规则引擎检测 | ⚠️ 基础实现 |
| **同行人分析** | 出行记录、同住同行 | 外部数据 | 🔲 预留 |

## 四、公司分析维度

> **核心目标**：排查外部供应商或关联公司是否存在向内部人员进行**利益输送**的迹象

### 4.1 资金流水总体规模

**目的**：评估公司业务规模和资金流动情况

| 指标 | 说明 | 实现状态 |
|------|------|----------|
| **累计进账** | 公司账户的总流入金额 | ✅ 已实现 |
| **累计支出** | 公司账户的总流出金额 | ✅ 已实现 |
| **交易笔数** | 统计期间的交易总数 | ✅ 已实现 |
| **账户数量** | 公司名下的银行账户数 | ⚠️ 部分实现 |

### 4.2 与调查单位的资金往来

**目的**：排查公司与"我单位"之间是否存在直接资金流动

| 指标 | 说明 | 判断逻辑 | 实现状态 |
|------|------|----------|----------|
| **往来金额** | 与调查单位的进出流水合计 | 单独统计 | ⚠️ 需配置 |
| **往来占比** | 往来金额 / 公司总流水 | 占比越高越需关注 | ⚠️ 需配置 |
| **往来明细** | 具体交易记录 | 逐笔核查 | ⚠️ 需配置 |

> 📌 **需要配置**：调查单位名称（关键词列表）

### 4.3 与关键人员的关联交易（利益输送排查）

**目的**：发现公司是否向调查单位人员进行利益输送

| 指标 | 说明 | 判断逻辑 | 实现状态 |
|------|------|----------|----------|
| **重名人员筛查** | 交易对象与核查人员名单比对 | 名称匹配 | ✅ 已实现 |
| **关联转账次数** | 与核心人员的转账笔数 | 次数越多越可疑 | ✅ 已实现 |
| **关联转账金额** | 与核心人员的转账总额 | 金额越大越需关注 | ✅ 已实现 |
| **资金流向** | 公司→个人 或 个人→公司 | 区分收付方向 | ✅ 已实现 |

### 4.4 现金交易分析

**目的**：发现潜在的隐匿资金或洗钱风险

| 指标 | 说明 | 判断逻辑 | 实现状态 |
|------|------|----------|----------|
| **有无现金收支** | 是否存在现金存取记录 | 是/否标记 | ✅ 已实现 |
| **现金交易笔数** | 现金交易的数量 | 统计 | ✅ 已实现 |
| **现金交易性质** | 提现/存现分类 | 详细分类 | ✅ 已实现 |
| **现金总额** | 现金收支合计 | 大额预警 | ✅ 已实现 |

> ⚠️ **风险信号**：公司账户大量提现但无合理用途说明

### 4.5 基础信息核查

**目的**：核实公司注册背景和账户信息

| 指标 | 说明 | 数据来源 | 实现状态 |
|------|------|----------|----------|
| **银行账户信息** | 账户列表、开户行 | 银行流水 | ⚠️ 部分实现 |
| **企业登记信息** | 注册资本、法人、股东 | 外部数据 | 🔲 预留 |
| **经营范围** | 主营业务 | 外部数据 | 🔲 预留 |
| **关联企业** | 股权关系、实控人 | 外部数据 | 🔲 预留 |

---

### 公司分析维度汇总表

| 维度 | 核心问题 | 当前支持 |
|------|----------|----------|
| 资金规模 | 公司有多少资金流动？ | ✅ |
| 调查单位往来 | 与我单位有无资金关系？ | ⚠️ 需配置单位名称 |
| 关键人员关联 | 有无向内部人员转账？ | ✅ |
| 现金交易 | 有无可疑现金操作？ | ✅ |
| 基础信息 | 公司背景是否可疑？ | 🔲 需外部数据 |

## 五、三类核心发现目标

报告通过上述维度的交叉比对，主要发现以下问题：

| 问题类型 | 判断逻辑 | 风险等级 |
|----------|----------|----------|
| **收支不抵** | 个人收入无法覆盖支出+大额消费 | 🔴 高 |
| **异常资金往来** | 与供应商/利益相关方的不明往来 | 🔴 高 |
| **资金来源不明** | 购房款/还款来源不清、第三方代付 | 🟠 中 |

## 七、数据复用铁律

> ⚠️ **核心原则**：报告生成严禁重新读取原始 Excel，必须复用已清洗/已计算的缓存数据。

| 数据项 | 数据来源 |
|--------|----------|
| 工资统计 | `profiles.json` → `salaryTotal`, `salaryRatio` |
| 现金交易 | `profiles.json` → `cashTransactions` |
| 理财持仓 | `profiles.json` → `wealthTotal` |
| 异常收入 | `derived_data.json` → `income` |
| 借贷分析 | `derived_data.json` → `loan` |
| 可疑交易 | `suspicions.json` |

## 八、审计视角补充分析维度

> 以下维度从纪检审计专业视角补充，部分需外部数据支持，可作为报告建议项。

### 8.1 个人/家庭报告补充项

| 补充维度 | 审计意义 | 数据需求 | 当前状态 |
|----------|----------|----------|----------|
| **时间线分析** | 资金异动与关键时间节点对照 | 职务变动、项目节点 | 🔲 需外部数据 |
| **购房资金溯源** | 每套房首付/还贷资金来源穿透 | 不动产+流水 | 🔲 需外部数据 |
| **大额消费事件** | 购车、装修、海外消费与收入对比 | 流水 | ⚠️ 可基于流水分析 |
| **异常规律性支出** | 定期向固定对象转账 | 流水 | ✅ 可复用借贷分析 |
| **关联企业穿透** | 被查人名下或实控企业 | 工商数据 | 🔲 需外部数据 |
| **家庭收支平衡表** | 全家收入 vs 全家资产+消费 | 流水+资产 | ⚠️ 需整合 |

### 8.2 公司报告补充项

| 补充维度 | 审计意义 | 数据需求 | 当前状态 |
|----------|----------|----------|----------|
| **合同对照** | 资金往来与合同金额对比 | 合同数据 | 🔲 需外部数据 |
| **时间节点关联** | 资金流入与中标时间对比 | 项目/中标时间 | 🔲 需外部数据 |
| **虚构交易识别** | 有资金往来但无实际货物 | 合同+物流 | 🔲 需外部数据 |
| **公司股权穿透** | 实控人是否与被查人关联 | 工商数据 | 🔲 需外部数据 |
| **员工名单比对** | 公司员工与转账对象比对 | 员工花名册 | 🔲 需外部数据 |

### 8.3 综合研判补充项

| 补充维度 | 审计意义 | 数据需求 | 当前状态 |
|----------|----------|----------|----------|
| **收入支出平衡表** | 汇总全家收入 vs 全家消费+资产 | 流水+资产 | ⚠️ 需整合 |
| **资金来源分类** | 合法/不明/可疑收入占比 | 流水 | ⚠️ 可基于分析结果 |
| **时间轴事件图** | 职务变动+重大交易+资金异动 | 流水+外部 | 🔲 需外部数据 |
| **关系图谱** | 人-公司-人 资金链路可视化 | 流水 | ✅ 已有图谱模块 |
| **问题等级分类** | 需核实/高度可疑/基本确认 | 分析结果 | ⚠️ 需添加逻辑 |

## 九、待完善典型场景

> 以下场景记录了审计分析中的典型需求，作为后续功能完善方向。

### 场景1：购房资金穿透

**问题**：甲某某2023年购房首付120万，但年工资仅15万

**报告应回答**：
- 首付120万的资金来源是什么？
- 是否有他人代付？代付人与被查人关系？
- 如果是"借款"，借款人是谁？是否有还款记录？

**数据需求**：不动产登记数据 + 银行流水

**当前状态**：🔲 待实现

---

### 场景2：时间节点关联

**问题**：A公司2024年3月中标某项目，2024年4月向甲某某转账5万

**报告应回答**：
- 这笔转账的时间与中标时间是否吻合？
- 甲某某与该项目是否有关联（评标委员、主管领导等）？
- 转账摘要是什么？是否合理？

**数据需求**：项目/中标时间数据 + 银行流水

**当前状态**：🔲 待实现

---

### 场景3：家庭收支平衡

**问题**：甲某某家庭总资产（房+车+存款）约500万，但夫妻二人年收入合计仅30万

**报告应回答**：
- 按现有收入，需要多少年才能积累这些资产？
- 是否存在"财产明显超过合法收入"的情况？
- 资产增长曲线与收入增长是否匹配？

**数据需求**：不动产 + 车辆 + 银行流水

**当前状态**：⚠️ 部分可实现（流水部分）
