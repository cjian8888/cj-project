## 📝 任务目标

增强银行账户识别和过滤能力,为后续的报告生成提供准确的银行账户数据。

### 涉及文件
- `data_cleaner.py` - 数据清洗模块
- `financial_profiler.py` - 资金画像模块

### 任务清单

#### 1.1 在 `data_cleaner.py` 中增加银行账户识别逻辑

**需要添加的功能**:
- [ ] 添加 `account_type` 列
  - 值: `借记卡` / `信用卡` / `理财账户` / `证券账户`
  - 判断逻辑:
    - 账号长度16-19位 → 借记卡/信用卡
    - 账号包含"基金"/"理财"/"证券" → 理财账户/证券账户
    - 开户行包含"基金"/"证券" → 对应账户类型

- [ ] 添加 `account_category` 列
  - 值: `个人账户` / `对公账户` / `联名账户`
  - 判断逻辑:
    - 根据账户名称或摘要判断

- [ ] 添加过滤非真实银行卡的逻辑
  - 过滤条件:
    - 账号长度异常(非16-19位)
    - 账号包含"基金"/"理财"/"证券"等关键词
    - 开户行为"XX基金"/"XX证券"
  - 添加 `is_real_bank_card` 布尔列

#### 1.2 在 `financial_profiler.py` 中新增账户提取函数

**需要添加的功能**:
- [ ] 新增 `extract_bank_accounts()` 函数
  - 功能: 从流水中提取唯一银行账户列表
  - 输入: 清洗后的流水数据
  - 输出: 去重后的银行账户列表
  - 字段包含:
    - 银行名称
    - 完整卡号(部分隐藏)
    - 账户类型
    - 账户类别
    - 账户状态(正常/停用)
    - 最后交易时间

- [ ] 账户去重逻辑
  - 同一账号只保留一条
  - 合并多银行信息(如果同一账号在不同银行出现)

- [ ] 账户状态判断
  - 根据最后交易时间判断是否正常/停用
  - 规则: 超过180天无交易 → 停用
