## 一、今日工作概述

### 1.1 核心问题修复（已完成）

**问题背景**: 报告中的收入数据严重失真
- 家庭概况显示工资收入为0（实际应为1088.97万）
- 显示总资金流入8442万（实际是流水，不是真实收入）
- 个人工资占比显示0.0%（实际应为70%左右）

**根本原因**:
1. 系统把理财/定存到期本金算作"收入"（这是本金返还，不是收入）
2. 系统只对冲了min(买入,赎回)，历史存量赎回未被剔除
3. 工资数据从错误的字段获取

**修复内容**:

| 修复项 | 修改文件 | 效果 |
|--------|---------|------|
| 修复真实收入计算 | `financial_profiler.py` | 新增剔除理财历史存量和定存到期 |
| 修复家庭工资汇总 | `investigation_report_builder.py` | 从yearly_salary正确获取工资 |
| 修复个人工资占比 | `investigation_report_builder.py` | 使用real_income计算占比 |
| 更新缓存数据 | `update_real_income_cache.py` | 重新计算所有实体真实收入 |

**关键数据修复结果**:

| 指标 | 修复前 | 修复后 | 变化 |
|------|--------|--------|------|
| 家庭工资收入 | 0万 | **1088.97万** | ✅ |
| 家庭真实收入 | 8442万（流水） | **1594万** | ✅ |
| 工资占比 | 0.0% | **68.3%** | ✅ |
| 施灵真实收入 | 2234万 | **901万** | ✅ |
| 施灵工资占比 | 28.6% | **70.9%** | ✅ |

## 四、关键技术信息

### 4.1 真实收入计算逻辑

```python
# 核心公式（financial_profiler.py）
real_income = (
    total_income                    # 原始流水收入
    - self_transfer_offset          # 自我转账对冲
    - wealth_principal_offset       # 理财本金对冲（当期）
    - wealth_historical_redeem      # 理财历史存量赎回
    - deposit_redemption            # 定存到期本金
    - loan_in                       # 贷款流入
    - refund_in                     # 退款
)
```

**施灵案例数据**:
- 原始流水收入：6903.58万
- 自我转账对冲：1786.99万
- 理财本金对冲：2834.90万
- 理财历史存量：1333.23万 ⭐新增剔除
- 定存到期：987.61万 ⭐新增剔除（但当前为0，需检查）
- 真实收入：901.54万

### 4.2 工资数据获取路径

```
正确路径:
profiles.json → yearly_salary → summary → total

错误路径（旧）:
profiles.json → salaryTotal （此字段可能为空或错误）
```

### 4.3 资产数据来源
