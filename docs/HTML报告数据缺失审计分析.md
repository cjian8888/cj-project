## 四、根因分析

### 4.1 数据流向断层

```
后端计算数据流向：
┌──────────────────────────────────────────────────────────────┐
│  1. 分析阶段 (analyzer)                                      │
│     salary_analyzer.py → 年度工资数据 ✅                     │
│     effective_expense_calculator.py → 消费分类 ✅            │
│     financial_profiler.py → 对手方统计 ✅                    │
└────────────────────┬─────────────────────────────────────────┘
                     │ 已存入 profiles.json / derived_data.json
                     ▼
┌──────────────────────────────────────────────────────────────┐
│  2. 报告构建阶段 (report_builder)                            │
│     investigation_report_builder.py → build_report_v5() ✅   │
│     数据已封装到 report["person_sections"]                   │
└────────────────────┬─────────────────────────────────────────┘
                     │ 已传递到模板渲染层
                     ▼
┌──────────────────────────────────────────────────────────────┐
│  3. 模板渲染阶段 (templates)                                 │
│     person_section.html → 未使用这些数据 ❌                  │
│     仅使用：总流入、总流出、房产、理财、银行存款基础数据      │
└──────────────────────────────────────────────────────────────┘
```

**结论**: 问题出在模板层 `templates/report_v3/person_section.html`，该模板未调用后端传递的详细分析数据。
