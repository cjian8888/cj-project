## 当前系统实现状态

### ✅ 已实现

| 模块 | 功能 | 数据来源 |
|------|------|----------|
| 工资识别 | 4轮识别算法 | `financial_profiler.py` |
| 现金交易 | 存取现统计 | `data_cleaner.py` → `is_cash` |
| 理财持仓 | 购买-赎回估算 | `financial_profiler.py` |
| 收支匹配 | 工资占比分析 | `profiles.json` → `salaryRatio` |
| 大额交易 | 阈值筛查 | `income_analyzer.py` |
| 借贷分析 | 双向往来/网贷 | `loan_analyzer.py` |
| 家庭识别 | 同户人/户籍人口 | `family_analyzer.py` |
| 公司→个人关联 | 直接转账检测 | `suspicion_detector.py` |
| 图谱可视化 | 资金流向图 | `flow_visualizer.py` |

### ⚠️ 部分实现 / 需增强

| 模块 | 现状 | 缺口 |
|------|------|------|
| 银行账户信息 | 有账号列表 | 缺余额、状态 |
| 与调查单位往来 | 无 | **需配置"调查单位"名称关键词** |
| 家庭资产汇总 | 有家庭识别 | 汇总统计逻辑需整合 |

### 🔲 需外部数据源

| 维度 | 说明 |
|------|------|
| 房产情况 | 需不动产登记数据 |
| 车辆情况 | 需车管所数据 |
| 企业登记信息 | 需工商数据 |
| 纳税记录 | 需税务数据 |
| 出行记录 | 需航空/铁路数据 |

## 缺失内容分析

### 1️⃣ 配置层面

| 缺失项 | 说明 | 建议 |
|--------|------|------|
| **调查单位名称** | 公司分析需筛查"与我单位"往来 | 在 `config.py` 添加配置项 |
| **大额阈值** | 当前硬编码 | 已有，需确认是否可前端配置 |

### 2️⃣ 数据整合层面

| 缺失项 | 说明 | 建议 |
|--------|------|------|
| **家庭资产汇总** | 需把 `family_finance.py` 整合到报告 | 调用 `calculate_family_total_assets()` |
| **公司报告模板** | 当前 `report_service.py` 只有个人 | 新增 `build_company_report()` |

### 3️⃣ 报告生成层面

| 缺失项 | 说明 | 建议 |
|--------|------|------|
| **JSON 输出** | 当前只有 HTML | 按准则格式输出 JSON |
| **家庭层级结构** | 当前是扁平的个人列表 | 改为 `family` + `member_details` |
| **综合研判** | 需自动汇总问题 | 遍历所有风险项生成清单 |
