# 穿云审计系统 - 问题排查与改进清单 (Issue Discovery)

## 1. 核心体验与流程问题 (High Priority)

### 1.1 关系图谱显示失效 (High Priority - Fixed Condition)
*   **现象**: 前端无法显示图谱，但后端生成的 HTML 效果良好。
*   **修复**:
    *   后端: 在 `api_server.py` 中强制调用 `flow_visualizer` 确保 HTML 生成。
    *   前端: 增加 "HTML Report" 视图模式 (Iframe)，并作为 Graph Data 加载失败时的备选项。
    *   **状态**: 已修复代码，待验证。

### 1.2 核心指标 (KPI) 展示信息过少 (High Priority - Fixed Condition)
*   **现象**: 指标过于基础。
*   **修复**:
    *   后端: 增强 `serialize_profiles`，增加现金总额、最大单笔交易等字段。
    *   前端: 重构 `KPICards`，展示“资金规模”、“现金占比”、“风险研判”等更有价值的审计指标。
    *   **状态**: 已修复代码，待验证。

### 1.3 数据一致性与“黑盒”感
*   **现象**: 点击分析后用户对正在处理的数据量、时间跨度一无所知。
*   **建议**: 增加“数据预检”步骤，告知用户识别到的文件数量和交易行数。

## 2. 功能完整性问题

### 2.1 报告与导出
*   **现状**: 报告列表存在，但用户反馈需要文件保存功能 (实际代码已在 `output/` 目录实现，但前端感知不强)。
*   **建议**: 前端增加明显的“打开报告文件夹”或“下载综合报告”按钮，打通 B/S 架构下的文件获取路径。

### 2.2 证据溯源 (Drill-down)
*   **现状**: 风险情报无法点击查看原始明细。
*   **建议**: 建立 风险项 -> 原始流水明细 的交互跳转。

## 3. UI/UX 细节问题
*   **空状态**: 图谱未生成时显示 "404"，给用户造成系统故障的误解。
*   **进度反馈**: 需要更细粒度的进度展示。
